# v0.5-keyword-variants — p@ssphrase / fl@g / s3cr3t 加入 keyword 白名单

## 背景

per Owner 2026-06-20 19:39 拍板: 实战累积再加 3 个 keyword 变种:
- `p@ssphrase` — `pass` 的常见变形 (p@ss + phrase)
- `fl@g` — `flag` 的常见变形 (f + l + @ + g)
- `s3cr3t` — `secret` 的 leetspeak 变形 (s3cr3t)

跟 v0.5-keyword-substring-journal (commit `7a80a56`) 的 8 个 keyword 一起,owner 高优先级 keyword 白名单
从 8 → 11 个。

## 目标

1. `core/suspicious.py:KEYWORDS` — 加 3 个 keyword
2. `core/utils/rule_scanner.py:_SENSITIVE_KEYWORDS` — **同步加 11 个 (上次 commit 漏修的 bug 一并补)**
   - 旧实现只有 5 个 (`key`, `flag`, `secret`, `ctf`, `password`),缺 `pass` / `f1ag` / `p@ssw0rd` / `p@ssphrase` / `fl@g` / `s3cr3t`
   - 这是上次"单一 source of truth 铁律"违反的 bug,违反场景: `has_sensitive_keyword("this_is_not_password")` 走 rule_scanner 走 `_SENSITIVE_KEYWORDS` 漏匹配(虽然 `password` 在内,但 `pass` 不在,且 rule_scanner 是 string in list 子串匹配,有 `password` 应该命中 — 但其他变种缺)
   - 修法: 全量替换成 11 个,跟 suspicious.KEYWORDS 同步
3. 测试 — `test_suspicious_keyword_substring.py` 加变种测试 + `test_rule_scanner.py` 加 11 keyword 同步测试
4. 注释同步 — 3 个文件里提到 keyword 列表的 docstring + 注释全部更新
