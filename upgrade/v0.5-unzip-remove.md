## 1. 背景

### 1.1 unzip 现状

- `unzip` Unix CLI，macOS/Linux 自带，**Windows 无原生版**（需 WSL/Git Bash 等包装）
- automisc macOS 时代为轻量 ZIP 列表/伪加密检测引入 unzip adapter（per `tools.md §3.5`）
- **Windows 端 100% 跑不通**：`shutil.which("unzip")` 返回 None → `executable not found` exit 127（per `v0.5-train-018` 实战验证）
- `tools.md:379` 已标 ❌ pending（"Win 可走 7z / Python `zipfile` 替代"）
- `v0.5-extend-tools-subdir-flexible.md` 行 120 明确说"Win 没装 ffmpeg/ffprobe/sox/**unzip**/tshark/tcpdump/grep, 等 v0.5+ 评估装"

### 1.2 为什么现在删（per Owner 2026-06-30 21:21 拍板）

- 项目 `AGENTS.md §2.3 v3.3` Windows only，unzip Win 不可用
- **替代方案齐备**：
  - `sevenz` adapter 已覆盖 ZIP 列表/提取/伪加密检测（且 Win 已装 7z 26.02）
  - `sevenz_extract` adapter 已覆盖真实解压（GUI 工具栏"📦 7z 解压"）
  - `zip_classify` 自研 Python 已覆盖真/伪加密判定（per `v0.5-zip-pseudo-per-entry-classify`）
  - `zip_chain.py:TryUnzipAction` 自研 Python `zipfile` 已覆盖无密码解压
- 留着 unzip adapter = **永远 127 + 误导用户 + 代码冗余**
- 跟 `v0.5-stegseek-remove.md` 治理一致性：**所有 Win 不可用 adapter 都要清**

### 1.3 删除后的能力矩阵

| 能力 | 现状 (unzip + 替代品) | 删后 (统一 sevenz/zipfile) |
|---|---|---|
| **ZIP 列表** (`unzip -l`) | unzip adapter + sevenz adapter | **sevenz adapter** (Win/macOS 全跑通) |
| **ZIP 完整性测试** (`unzip -t`) | unzip adapter + sevenz adapter | **sevenz adapter** |
| **伪加密检测** (unzip stderr 信号) | unzip adapter + sevenz adapter + zip_classify | **sevenz adapter** (报"cannot open as archive") + **zip_classify** (per-entry 判定) |
| **ZIP 真解压** | try_unzip action (Python zipfile) | **try_unzip action 保留**（自研 zipfile） |
| **ZIP 字典爆破** | john adapter + zipcrack | **保留**（跟 unzip 无关）|
| **hex 命中 zip magic 后的 follow-up 解压** | `hex_router._run_unzip` 调 unzip.exe | **`hex_router._run_7z`** 调 sevenz (per Owner 21:21 Q2=A) |

**Net 变化**：
- ✅ Win 端 auto-run 不再跑 unzip（少 1 个 127 FAIL，journal 干净）
- ✅ GUI 工具栏去掉 dead entry（unzip menu 显示但跑不通）
- ✅ 代码净简化（删 1 个 adapter 文件 + 1 个 hex_router fallback 函数 + 5 处注册/推荐表/menu/auto-run 引用）
- ⚠️ 失去 macOS 端 unzip 报错的"End-of-central-directory signature not found"强信号 → **sevenz 报"cannot open as archive"同义**，能力等价
