# v0.5-tmp-text-mode-2 — QFileDialog 只在 decoder 真写文件时弹

> **创建时间**：2026-06-14 12:44
> **状态**：✅ done (commit (待 push))
> **触发**：Owner "编码转换、进制转换不应该弹出 QFileDialog. 比如我刚刚输入 28372c37290a..., 然后点击 Hex->ASCII, 竟然弹出了 QFileDialog, 这是不合理的. 只有有文件输出时才弹出"

---

## 1. 问题

v0.5-tmp-text-mode (12:05) 把**所有 text-based decoder** 都弹 QFileDialog：
- ✅ coords-qr（写 PNG 文件）—— 弹合理
- ❌ hex-ascii（不写文件，result 是 string）—— 弹**不合理**

Owner 12:44 明确：**只有 decoder 真有文件输出时才弹 QFileDialog**。

---

## 2. 修复

### 2.1 main_window `_run_decoder` 判定

**之前**（v0.5-tmp-text-mode）:
```python
if is_text_based:  # 所有 text-based decoder 都弹
    out_dir = QFileDialog.getExistingDirectory(...)
    ...
```

**现在**（v0.5-tmp-text-mode-2）:
```python
if is_text_based:
    candidate = self._extract_input_candidate()
    if not candidate:
        return  # 早返

    # v0.5-tmp-text-mode-2: 仅当 decoder 真写文件才弹 QFileDialog
    spec = get_decoder(decoder_name)
    import inspect
    sig = inspect.signature(spec.run)
    produces_file = "output_dir" in sig.parameters  # 签名含 output_dir = 写文件

    out_dir = None
    if produces_file:
        out_dir = QFileDialog.getExistingDirectory(...)
        if not out_dir:
            return  # 用户取消
```

**判断标准**：`spec.run` 函数签名含 `output_dir` 参数 → 写文件 → 弹 dialog
- `hex-ascii` (base_convert.py) `_runner(file_path=None, text=None, **_)` → **不含 output_dir** → 不弹
- `coords-qr` (coords_to_qr.py) `_runner(file_path=None, text=None, output_dir=None, **_)` → **含 output_dir** → 弹
- `base64-image` (base64_image.py) `decode_file_to_image(file_path, *, output_dir=None, ...)` → 含 output_dir（但走 file 模式，不走 text-based 分支）

### 2.2 test 调整

**移除**：`test_menu_hex_ascii_uses_input_not_current_file` / `test_menu_hex_ascii_empty_input_no_current_file_read` 的 monkey-patch QFileDialog（不需要了）
- hex-ascii 不弹 dialog → 不需要 patch

**新增**：`test_hex_ascii_no_qfiledialog_no_write_file`
- 故意**不** monkey-patch QFileDialog
- 跑 hex-ascii 后验证 QFileDialog.getExistingDirectory 没被调用
- 同时 (7,7) 应在 output
- 防以后回归（有人把 hex-ascii 也加进 produces_file 列表）
