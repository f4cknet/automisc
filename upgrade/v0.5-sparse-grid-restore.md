# v0.5-sparse-grid-restore — sparse (col, char) 网格 → 等宽 ASCII 字符画

> **背景**: Owner CTF 实战 (2026-07-01) 反序列化 Python Pickle 拿到 30×76 三态 sparse grid
> (`enc=[[(3, 'm'), ...], ...]`) — 现有工具 (file/strings/binwalk/exiftool/...) 全部不适配
> 这个 sparse 2D 表示, 实战中只能手撕 Python 写 temp array 还原. Owner 拍板:
> "把稀疏 (col_index, char) 展开成等宽 ASCII 字符画, 这个能抽象成工具吗?
>  不管数组内容怎么变, 都可以用, 比如 (3, 'm'), (4, '"') 改为 (4, 'm'), (3, '#')"

---

## 1. 需求

### 1.1 解决什么问题
CTF 实战中, **sparse 2D 网格** 类 challenge 输出形式各异:
- **Minesweeper 完整状态 dump**: `[(1, '#'), (3, '"')...]` 三态 (雷 / 已翻 / 标旗)
- **数织图 (Nonogram / Picross) hint 序列化**: `[(index, color)]`
- **Hashiwokakero (桥 / Bridges) 状态**: 岛屿 / 空地 / 桥标记
- **ASCII 拼图**: 把稀疏坐标还原成可视字符画直接看 flag 字样
- **Tapa / Slitherlink / Akari**: 三态墙 / 空 / 标记

每一道题都要手撕 Python — **重复劳动**, 该抽象成工具.

### 1.2 答案形式 (per 用户原话)
> "把这个抽成工具, 不管数组内容怎么变, 都可以用"

- 输入: sparse 2D 网格 (list of list of `(col_index, char)` tuples)
- 输出: 等宽 ASCII 字符画, 一目了然

---

## 2. 决策

### 2.1 走 `core/decoders/` 路线 (不是 `tools/shared/`)
**理由**: sparse grid → ASCII 是**纯 stdlib 计算**, 跟 `hex-ascii` / `magic_sniffer` / `pyc_decompiler` 完全同模式.
无外部 binary 依赖, 不需要 `tools/base.py:ToolAdapter`, 走 `DecoderSpec` 注册.
(per AGENTS §3.5 "GUI 工具来源说明 — tools/ vs decoders/")

### 2.2 命名 + GUI 分类 (per Owner 2026-07-01 拍板 4 决策)
| Q | 决策 |
|---|---|
| Q1 命名 | A: `🧩 稀疏网格还原` (内部 `name="sparse_grid_restore"`) |
| Q2 GUI 分类 | **塞入"共享基础工具 (PR1)"**, 跟 `file`/`strings`/`binwalk`/`foremost`/`exiftool`/`xxd`/`trid` 同级 |
| Q3 进 auto-run | ❌ 不进 (跟 trid 同节奏, 跟 AGENTS §1 铁律 7 一致) |
| Q4 实战 fixture | ❌ 不 commit 实战数据 (`enc=[[...]]` 30×76 grid 留 Owner 本地 `Challenge/` 目录, 不入仓) |

### 2.3 设计选型
- **3 种输入格式 auto-detect** (跟 hex-ascii 同思路):
  1. JSON: `[[3,"m"],[4,"\""],...]`
  2. Python literal: `[(3, 'm'), (4, '"'), ...]`
  3. 格式化 raw text (如 '随波逐流' 反序列化工具 dump): `[(3, 'm'), (4, '"'), ...]` (跟 Python literal 同处理, ast.literal_eval 兜底)
- **`text_only=True`**: 跟 hex-ascii / cipher_decoders 同模式. GUI 工具栏触发后从 input 区 (左上角文本框) 拿 text, 不读 file_path. (per `fix_decoder_registry_pyc_magic` + `v0.5-cipher-decoders-textfix`)
- **不写文件**: 只渲染到 GUI output 区 + CLI stdout. (per AGENTS §1 铁律 7)
- **size guard**: `MAX_CELLS=100_000` (100K cells ≈ 1MB 字符), `MAX_INPUT_SIZE=10MB` text
- **CSR dense 化算法**:
  1. 推断 col 宽 = max(col index) - offset + 1 (offset=1 默认, 跟原 writeup 一致)
  2. 每行建 `[default_char] * cols` 模板
  3. 遍历该行的 (col, char) tuples 覆盖到对应位置
  4. last-wins (跟原 writeup `temp[t[0]]=t[1]` 行为一致; 同 col 后出现的覆盖先出现的)
- **stats 输出**: `{rows, cols, total_chars, empty_rows, max_col}` (便于 GUI 显示, 也给决策: empty_rows > 50% 提示"这 grid 是否有效?")

### 2.4 不动什么 (per AGENTS §6 架构优先 + §5.2 防单题打补丁)
- ❌ 不挂 auto-run 池 (跟 trid 同节奏)
- ❌ 不调 binary, 不动 `extend-tools/manifest.yaml` / `install.ps1`
- ❌ 不写专用 fixture 进 `tests/fixtures/` (per Q4)
- ❌ 不改 `tools/base.py` / `core/registry.py` / `tools/__init__.py` (走 decoder 路径, 不走 adapter)
- ❌ 不动现有 decoder (`base_convert` 等)

## 3. 设计

### 3.1 decoder 主体 (估算 ~250 LOC)
```
src/automisc/core/decoders/sparse_grid_restore.py
├── class SparseGridError(Exception)     # 解析/渲染失败
├── class SparseGridResult(dataclass)     # input / detected_format / rendered / stats / errors
├── MAX_CELLS: int = 100_000
├── MAX_INPUT_SIZE: int = 10 * 1024 * 1024
├── _detect_format(text) -> str            # "json" / "python-literal" / "raw-text" / "empty"
├── _parse_input(text, fmt) -> list[list[tuple[int, str]]]
├── render(rows, *, default_char, sep, one_based, keep_order) -> (list[str], dict)
├── restore(text) -> SparseGridResult      # 总入口
└── _register()                            # 注册到 DecoderSpec (跟 base_convert §419-507 同模式)
```

## 6. 风险与边界

| 风险 | 兜底 |
|---|---|
| 超大 grid OOM (e.g. col > 1000 / rows > 500) | `MAX_CELLS=100_000` size guard 报错 |
| 嵌套层数 > 2 (e.g. `[[[...]]]`) | `_parse_input` row[ri] must be list 检查 |
| tuple 含奇怪 char (e.g. 多字符字符串) | `len(char) != 1` 报错 |
| tuple 含非 int col (e.g. string col) | type check 报错 |
| 跨平台 | Win only (per v0.5-windows-only 治理), 但 stdlib 跨平台 macOS Linux 也能跑 |
| 离线 | 纯 stdlib, 0 网络调用, 守 STRUCTURE §1 完全离线铁律 |
| 同 decoder name 撞车 (unlikely) | registry 是 list (注册顺序), 唯一靠 name |
| GUI input 过大 | `MAX_INPUT_SIZE=10MB` 兜底; 实操粘贴 30×76 grid text <10KB, 远低于 |
| Owner CTF enc 数据 commit 进仓 | ❌ 拒绝 (per Q4 决策), `tests/fixtures/.gitignore` 不动 |
