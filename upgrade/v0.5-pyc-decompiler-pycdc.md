### 1.2 实施路径 (per v0.5-pyc-decompiler-pycdc spec 现场)

**实际走过的路径** (per 2026-07-01 10:08-10:30 实战):

1. **找 pycdc Win prebuilt binary**: PyPI `pycdc 0.1.0` 是同名礼品 API client (无关注释放), `tahmidrayat/pycdc-windows` 仓库有 GitHub Actions 但无 release 标签, winget/scoop 库空, CSDN 要 C 币 → **找不到现成 prebuilt**
2. **决定自己编译**: 装 cmake 4.3.3 (winget Kitware.CMake) + msys2 (winget MSYS2.MSYS2) + MinGW64 toolchain (pacman -S mingw-w64-x86_64-toolchain, 配清华 tuna 镜像)
3. **git clone zrax/pycdc**: `git clone --depth=1 https://github.com/zrax/pycdc.git`
4. **cmake 编译**: MinGW64 + 静态链 libgcc/libstdc++/winpthread → pycdc.exe 4.2MB (避免 `STATUS_DLL_NOT_FOUND`)
5. **验证 flag.pyc**: 输出完全正确, ciphertext 真实字符串 `['96', '65', '93', '123', ...]`, 无末尾 return artifact ✅
6. **装到 extend-tools/bin/win-x64/**: pycdc.exe + pycdas.exe (3.5MB, 字节码反汇编)

## 2. 目标

### 2.1 能力 A: Py2.x pyc 反编译优先走 pycdc

**目标**: `pyc_decompiler.py:run_pyc_decompiler` Py2.x 路由优先级:
1. **pycdc** (subprocess, 走 `tools/paths.py:resolve_tool_binary("pycdc")`)
2. **uncompyle6 + fix** (fallback, per 现状)
3. **dis** (last fallback, 字节码反汇编)

**Py3.x 路由**: 不变, 走 decompyle3 → dis fallback (pycdc 也支持 Py3.x 但我们不调, 跟现状一致; pycdc 实际 Py3.x 反编译跟 Py2.x 一样准, 后续实战触发再考虑 Py3.x 走 pycdc)。

### 2.2 能力 B: 跨平台 binary 部署 (per v0.5-platform-extend-tools)

- **Windows**: extend-tools/bin/win-x64/pycdc.exe (4.2MB, 静态链 libgcc/libstdc++/winpthread, 不需要额外 DLL)
- **macOS**: Homebrew 优先 (`brew install pycdc` — Homebrew 维护的 zrax/pycdc fork)
- **manifest.yaml** 加 pycdc entry (URL 空, sha256 填 owner 编译的 hash, notes 写"手工 build, 命令见 install.ps1 §Stage 1.5" 或 spec 附录)
- **install.ps1** 加 pycdc entry: url 空 → 跳过下载 → notes 提示要手工 build (跟 msvcr120 模式一致: "走 manifest + install.ps1 复用, 不下载")

### 2.3 不在本次范围

- ❌ **不动 Py3.x 路由** (decompyle3 仍为 Py3.x 首选, pycdc 不参与)
- ❌ **不删 uncompyle6 依赖** (fallback 需要, 实战 pycdc 失败时降级)
- ❌ **不删 fix_uncompyle6_consts_bug** (fallback 路径仍需要修 uncompyle6 已知 bug)
- ❌ **不改 PycDecompileResult 字段** (output_path 命名 `__pyc_py2.py` 不变, 跟 v0.5-pyc-decompiler-buttons 一致)
- ❌ **不改 GUI 工具栏** (3 按钮已 work, pycdc 是底层实现替换, 跟 GUI 入口解耦)
- ❌ **不写新 adapter** (pycdc 不走 auto_run / find_suspicious, 纯 decoder 内部依赖, 跟 pycdc-decompiler 一起自动调用)
- ❌ **不引 GUI menu entry** (per v0.5-sevenz-toolbar-cleanup "用于自动化探测的工具,可不显示在 GUI menu")
- ❌ **不暴露 pycdas** (字节码反汇编, GUI 没需求)

### 3.2 改 `core/decoders/pyc_decompiler.py`

```python
def _decompile_with_pycdc(file_path: str) -> tuple[str, str]:
    """Py2.x 反编译 (per v0.5-pyc-decompiler-pycdc), 走 pycdc C++ 反编译器.

    比 uncompyle6 准确 (独立 C++ 实现, 无 uncompyle6 已知 consts 索引 / lambda / closure bug).

    Args:
        file_path: .pyc 文件路径

    Returns:
        (source_code, "pycdc") - 成功
        ("", "pycdc") - 失败 (让 caller fallback 到 uncompyle6+fix)
    """
    from automisc.tools.paths import resolve_tool_binary
    pycdc = resolve_tool_binary("pycdc")
    if not pycdc:
        return "", "pycdc"
    try:
        result = subprocess.run(
            [pycdc, file_path],
            capture_output=True, text=True, timeout=30,
        )
        if result.returncode == 0 and result.stdout.strip():
            return result.stdout, "pycdc"
    except (subprocess.TimeoutExpired, FileNotFoundError, OSError):
        pass
    return "", "pycdc"


# run_pyc_decompiler Py2.x 路由改:
if is_py2 or force_version == 2:
    # Py2.x 优先 pycdc
    source, method = _decompile_with_pycdc(file_path)
    if not source:
        # fallback: uncompyle6 + fix (per v0.5-train-019)
        try:
            source, method = _decompile_with_uncompyle6(file_path)
        except Exception as e:
            err = f"uncompyle6 fallback failed: {e}"
    if not source and err is None:
        err = "pycdc + uncompyle6 都失败"
```

**注**: pycdc 路径下不走 `_fix_uncompyle6_consts_bug` (pycdc 无 bug, 跟 uncompyle6 不同的输出格式, 没必要套用 fix)。`_decompile_with_uncompyle6` 内部仍调 fix (uncompyle6 bug 历史包袱)。

## 6. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| pycdc binary 缺失 (手工 build 失败) | resolve_tool_binary 返回 None → fallback uncompyle6+fix (现状 work, 0 回归) |
| pycdc subprocess 超时 (复杂 pyc) | subprocess timeout=30s, 超时 fallback uncompyle6 |
| pycdc 不支持某 Py3.x magic (e.g. Py3.13+) | Py3.x 路由**不**走 pycdc, 走 decompyle3 (现状) |
| pycdc 输出格式跟 uncompyle6 不同 (注释头 / 缩进) | output_text 直接展示, 不做统一化 (per _on_decoder_finished 已经按字段打印) |
| install.ps1 加 skip_if_url_empty switch case 跟其他 case 冲突 | 独立 case, 不影响其他 tool (exiftool_zip / sevenz_extract / file_zip) |
| 国内网络装 cmake + msys2 + MinGW64 慢 | Tuna 镜像已配 (per 实操), 1-2 分钟 OK |
| pycdc 自编译 cmake GCC 报 `Permission denied C:\Windows\` | 设 TMP/TEMP = `C:\Users\zmzsg\AppData\Local\Temp` (per 实操) |
