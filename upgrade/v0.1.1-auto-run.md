## 设计

1. `gui/auto_runner.py` 新增 `AutoRunner(QThread)` 类
   - 持有 `core` + `recommendations` + `file_path` + `max_tools=5`
   - 串行跑（避免并发抢资源 + GUI 显示可读）
   - 4 signals：tool_started / tool_finished / chain_finished / chain_failed
   - 传完整 `ToolResult` 给 GUI（避免重复跑）
   - `stop()` 中途打断
2. `gui/main_window.py` 集成
   - `__init__` 加 `_auto_run_enabled=True` + `_current_recommendations=[]`
   - `dropEvent` 拖完文件后**自动启 AutoRunner**
   - 4 个 callback：started / finished / chain_finished / chain_failed
   - **状态栏实时显示进度**：`[auto 2/5] running strings on sample.txt…`
   - **Run 菜单**加 "Auto-run on drop" toggle
3. 错误处理：
   - `AutomiscError`（如 `ToolNotFoundError`）→ 链失败 + 终止后续
   - 未知异常 → 链失败 + 终止
4. 测试：
   - `test_auto_runner.py`：8 单测（chain / filter / error / stop）
   - `test_auto_run_on_drop.py`：4 e2e 单测（drop text / drop PNG / 关闭 / journal 累积）

## 关键技术决策

- **串行 vs 并发**：v0.1 简化用串行（可读 + 顺序显示 + 避免抢资源）。v0.5-8 改 QThreadPool 并发。
- **ToolResult 传递**：AutoRunner 调 `core.run_tool` 后**不重新跑**，直接 `tool_finished.emit(tool_name, summary, result)`。GUI 拿 result.stdout / result.suspicious_points 直接显示。
- **最大数 max_tools=5**：平衡"自动跑一轮"vs"不要把 GUI 塞满"。可在 MainWindow 改。
- **默认开启 auto-run**：Owner 反馈核心痛点是"还要手动点"，所以默认开。Run 菜单 toggle 关闭。

## 已知限制（v0.5+ 优化）

- **串行跑**慢（每个工具等前一个跑完）。v0.5-8 改 QThreadPool 并发（限制 3 并发）
- **score=0 的工具被过滤**（合理但工具可能"无所谓 score"）。v0.5-6 router 增强时统一考虑
- **没有进度条**（v0.1 简化）。v0.5-5 GUI 增强时加
- **拖新文件**打断旧 auto-run（用 `stop() + wait(2000)`）。简单可靠，没考虑 race condition（v0.1 简化足够）
