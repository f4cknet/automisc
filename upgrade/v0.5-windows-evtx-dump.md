| install.ps1 | ❌ 不管（brew / 软链手工） | ✅ 跟 exiftool / 7zr / foremost 同一套 Stage 1 |

### 1.2 为什么要装 Rust

Owner 要求 install.ps1 加 Rust 装步骤。**当前 evtx_dump 不需要 Rust**——GitHub release 直接发 `.exe` 单文件。

Rust 装的价值是**架构层面**的：

1. **未来 cargo install 兜底**：如果某个工具只有源码 release（per AGENTS.md §1 铁律 6 架构优先，**必须能自己编译**），Rust 装上才能走 `cargo install` / `cargo build`
2. **binwalk v3 备选**：binwalk v3 是 Rust 重写，per `manifest.yaml pip_packages.binwalk.notes` 当前用 pip v2.3.x 凑合；如果 v3 路线要走（性能 + 维护性），需要 Rust
3. **ad-hoc 写 adapter / CLI 工具**：CTF 应急实战经常需要快速撸个工具，Rust 装了能本地编译

**架构判定（per 铁律 6）**：✅ 能力升级不是单题补丁 → 走架构层。

## 4. Rust 装策略（Stage 0 新增）

### 4.1 工具选择

**rustup-init.exe**（官方入口）：
- URL: `https://win.rustup.rs/x86_64`
- 优点：官方、跨 shell、自动加 PATH、自动 cc 链接
- 缺点：~250MB（toolchain + cargo + rustc + std docs），首次装慢

**winget**（备选）：
- `winget install --id Rustlang.Rustup`
- 优点：系统包管理
- 缺点：依赖 winget 是否装（Win 11 默认有，Win 10 可能没有）

**决定**：rustup-init.exe 主路径，winget 备选（如果 rustup 下载失败）。

### 4.2 idempotent 检查

```powershell
# 检测 rustc 是否已装
$rustc_ok = $false
try {
    $ver = & rustc --version 2>$null
    if ($LASTEXITCODE -eq 0 -and $ver -match "rustc \d+\.\d+\.\d+") {
        $rustc_ok = $true
    }
} catch {}
```

如果已装 → skip；没装 → 下载 rustup-init + `-y --default-toolchain stable --profile minimal` 装。

**minimal profile** 跳过 rust-docs（省 ~200MB，CTF 应急不需要 std docs 浏览器）。

### 4.3 PATH 影响

rustup 装完会把 `%USERPROFILE%\.cargo\bin` 加到 PATH，**当前 shell 进程不会自动看到**。

**fix**（per Owner Windows 11 经验）：

```powershell
$env:Path = [System.Environment]::GetEnvironmentVariable('Path','Machine') + ';' + [System.Environment]::GetEnvironmentVariable('Path','User')
```

放 Stage 0 末尾，**让 install.ps1 后续 Stage 1 / Stage 2 步骤能立刻用 `cargo` / `rustc`**。

### 4.4 失败处理

- rustup 下载失败 → warning，continue（不阻塞后续 Stage）
- rustup 装失败 → warning，continue（同上）
- 这两个 warning 不算 $fail_count（因为 Rust 是 optional 加固，不是 Stage 1 / Stage 2 必需）

## 5. install.ps1 改动详解

### 5.1 Stage 0: Rust（新增）

```powershell
Write-Host "--- Stage 0: Rust toolchain (optional, for cargo install / binwalk v3) ---" -ForegroundColor Cyan

$rust_installed = $false
try {
    $ver = & rustc --version 2>$null
    if ($LASTEXITCODE -eq 0) {
        Write-Host "[skip] rustc already installed: $ver" -ForegroundColor Gray
        $rust_installed = $true
    }
} catch {}

if (-not $rust_installed) {
    try {
        Write-Host "[download] rustup-init.exe ..." -NoNewline
        $rustup = Join-Path $StageDir "rustup-init.exe"
        Download-File -Url "https://win.rustup.rs/x86_64" -Out $rustup
        Write-Host " OK" -ForegroundColor Green

        Write-Host "[install] rustup-init.exe -y --default-toolchain stable --profile minimal ..." -NoNewline
        & $rustup -y --default-toolchain stable --profile minimal | Out-Null
        if ($LASTEXITCODE -ne 0) { throw "rustup-init exit $LASTEXITCODE" }
        Write-Host " OK" -ForegroundColor Green

        # Refresh PATH so cargo / rustc visible
        $env:Path = [System.Environment]::GetEnvironmentVariable('Path','Machine') + ';' + [System.Environment]::GetEnvironmentVariable('Path','User')
        Write-Host "[rust] cargo bin: $env:USERPROFILE\.cargo\bin (refreshed PATH)"
    } catch {
        Write-Host " FAILED" -ForegroundColor Yellow
        Write-Host "         warning: $_" -ForegroundColor Yellow
        Write-Host "         Rust not installed; install.ps1 will continue." -ForegroundColor Yellow
    }
}
```

### 5.2 Stage 1: evtx_dump 条目（新增）

加到 `$binaries` 数组尾部：

```powershell
@{
    name = "evtx_dump"
    version = "0.12.2"
    # omerbenamram/evtx GitHub release, Win x86_64 prebuilt
    # 单文件 .exe (2.5MB, static link, 零运行时依赖)
    url = "https://github.com/omerbenamram/evtx/releases/download/v0.12.2/evtx_dump-v0.12.2.exe"
    target = "evtx_dump.exe"  # 简化文件名 (strip version suffix)
    post_extract = $null  # 单文件, no extract
}
```

**target 简化策略**：源文件名是 `evtx_dump-v0.12.2.exe`（含版本号），下载后 `Move-Item $tmp $dest -Force` 会用 `evtx_dump.exe`（去掉版本号），跟 `7zr.exe` / `foremost.exe` / `exiftool.exe` 命名一致。
