## 1. 背景与动机

### 1.1 Owner 实战 (2026-06-27)

`automisc` 在 macOS 上 `automisc-gui` 直接跑就行，因为 Homebrew 装的 binwalk / exiftool / 7z / foremost / zsteg / steghide / ... 全在 `/usr/local/bin` 或 `/opt/homebrew/bin`，`subprocess.run` 直接 `shutil.which` 命中。

Owner 在 Windows 11 + PowerShell 拉仓库装依赖跑 GUI：
- ✅ PySide6 跨平台，GUI 起来
- ✅ 9 个 adapter 用 Windows 内置工具（strings / file / xxd / grep / unzip / tcpdump / vol3 / tshark / ffmpeg）可用
- ❌ 4 个核心 adapter 报"找不到"：**binwalk / exiftool / 7z / foremost**
- ❌ 2 个不可用：**zsteg**（Ruby gem）/ **steghide**（无官方 Windows 版）

### 1.2 Owner 决策 (2026-06-27 Y)

| 决策点 | 拍板 | 备注 |
|---|---|---|
| steghide | 不装（**无自研替代**；Windows 上 JPEG/WAV 隐写真跑不了） | GUI 灰掉 + 文档说明 |
| zsteg | 不装（用自研 `lsb_detect` 替代） | v0.5-lsb-detector 已替代 auto-run 位 |
| macOS 兼容策略 | **A：双路径兼容**（macOS 优先 `which` → fallback extend-tools/） | 不破坏现有 macOS 体验 |
| 治理变更 | **走 §8 标准流程**（提案 + 改 §2.3 + §9 变更日志） | Owner 自审后 commit |

### 1.3 关键约束

- **AGENTS.md §2.3 macOS only** — 本 spec 是治理变更，**必须改 §2.3** 才合规
- **AGENTS.md §2.1 粒度** — 单 PR ≤ 400 行；不跨任务 ID
- **完全离线** — automisc 产品定位 (`prd.md §3.3`)，但 **install 脚本首次下 binary 是开发期工具**，不违反产品定位（用户跑 GUI 仍离线）

| 工具 | Windows binary 来源 | 大小 | 装 |
|---|---|---|---|
| **binwalk** | [ReFirmLabs/binwalk GitHub Releases](https://github.com/ReFirmLabs/binwalk/releases) `binwalk.exe` | ~5MB | ✅ |
| **exiftool** | [exiftool.org](https://exiftool.org) standalone Windows exe (renamed from exiftool(-k).exe) | ~10MB | ✅ |
| **7z (7zr.exe)** | [7-zip.org](https://www.7-zip.org) 独立版 (不需安装器) | ~1MB | ✅ |
| **foremost** | [raddyfiy/foremost](https://github.com/raddyfiy/foremost) `binary/foremost.exe`（**第三方 fork**，CTF 圈通用） | ~1MB | ✅ |

| **steghide** | 官方无 Windows 版 | — | ❌（无替代） |
| **stegseek** | 官方无 Windows 版（steghide 替代品） | — | ❌（跟 steghide 一起放弃） |

**总 Windows binary 体积**：~17MB，仓库不进 git（用 manifest.yaml + install 脚本）

### 2.2 macOS 兼容：双路径

```python
def resolve_tool_binary(name: str) -> str | None:
    """先 PATH → 再 extend-tools/bin/<platform>/"""
    # 1) PATH 优先 (macOS Homebrew / Windows 系统 PATH)
    found = shutil.which(name)
    if found:
        return found
    # 2) extend-tools fallback (Windows 没有 brew, 用这里的 binary)
    plat = "win-x64" if sys.platform == "win32" else "macos"
    candidate = EXTEND_TOOLS_BIN_DIR / plat / f"{name}.exe" if sys.platform == "win32" else EXTEND_TOOLS_BIN_DIR / plat / name
    if candidate.exists():
        return str(candidate)
    return None
```

**macOS 用户体验不变**：Homebrew 装过的工具仍然走 PATH（extend-tools/bin/macos/ 是空的）

### 2.3 不可用工具的 GUI 表现

**steghide / zsteg / stegseek 在 Windows 上 unavailable**：
- 菜单项前面 marker 从 `✓` 变成 `✗`（已有机制）
- 点击执行 → ToolResult 报 `exit_code=127` + stderr `"executable not found: steghide"`
- GUI 输出区显示"该工具在 Windows 不可用：JPEG/WAV 有口令隐写 v0.5+ 评估"

**机制**：增强 `menu_dock.py` ✓/✗ 判定，从"是否注册"改为"是否注册 **且 binary 可执行"**

### 3.2 `src/automisc/tools/paths.py` 设计

```python
"""extend-tools/ 路径解析 + 跨平台 binary 查找.

v0.5-platform-extend-tools: Owner 2026-06-27 拍板 Windows 跑 automisc.

逻辑:
1. PATH 优先 (macOS Homebrew / Windows 系统 PATH / 用户自装)
2. extend-tools/bin/<platform>/ fallback (Windows 上 GitHub release 下的 binary)

不缓存: 每次调用都重新查 (PATH 可能装新工具, 安装脚本刚跑完想立即生效).
"""
from __future__ import annotations
import shutil
import sys
from pathlib import Path

# 项目根 = extend-tools/ 的父目录
_EXTEND_TOOLS_DIR = Path(__file__).resolve().parents[3] / "extend-tools"
EXTEND_TOOLS_BIN_DIR = _EXTEND_TOOLS_DIR / "bin"
EXTEND_TOOLS_MANIFEST = _EXTEND_TOOLS_DIR / "manifest.yaml"

_PLATFORM_DIR = {
    "win32": "win-x64",
    "darwin": "macos",
    "linux": "linux-x64",
}.get(sys.platform, "unknown")


def platform_subdir() -> str:
    """当前平台的 extend-tools/bin/ 子目录名."""
    return _PLATFORM_DIR


def exe_suffix() -> str:
    """Windows 加 .exe, 其他平台空字符串."""
    return ".exe" if sys.platform == "win32" else ""


def resolve_tool_binary(name: str) -> str | None:
    """先 PATH → 再 extend-tools/bin/<platform>/.

    Args:
        name: 工具名 (e.g. "binwalk" / "7zr" / "tshark")

    Returns:
        绝对路径字符串 or None (找不到)
    """
    found = shutil.which(name)
    if found:
        return found
    if _PLATFORM_DIR == "unknown":
        return None
    candidate = EXTEND_TOOLS_BIN_DIR / _PLATFORM_DIR / f"{name}{exe_suffix()}"
    return str(candidate) if candidate.exists() else None


def list_available_tools() -> list[str]:
    """扫描 extend-tools/bin/<platform>/ + manifest.yaml 注册的工具名清单."""
    sub = EXTEND_TOOLS_BIN_DIR / _PLATFORM_DIR
    if not sub.exists():
        return []
    return [p.stem for p in sub.iterdir() if p.is_file()]
```

### 3.3 `src/automisc/tools/base.py` 改动

**改 1**：`_run_subprocess` PATH 跨平台（**修 Windows 上 macOS Homebrew 路径加错位置的 bug**）

```python
# 旧 (line 128-131):
homebrew_paths = "/opt/homebrew/bin:/usr/local/bin"
current_path = env.get("PATH", "")
if homebrew_paths not in current_path:
    env["PATH"] = f"{homebrew_paths}:{current_path}"

# 新:
if sys.platform == "darwin":
    # macOS: 追加 Homebrew 路径 (Apple Silicon + Intel)
    homebrew_paths = "/opt/homebrew/bin:/usr/local/bin"
    current_path = env.get("PATH", "")
    if homebrew_paths not in current_path:
        env["PATH"] = f"{homebrew_paths}:{current_path}"
# Windows / Linux: 不动 (用系统 PATH + venv Scripts/)
```

**改 2**：`check_available()` 默认走 `resolve_tool_binary`

```python
def check_available(self) -> bool:
    """检查外部工具是否可用.
    
    - binary_path 显式设置: 检查文件是否存在
    - 否则走 resolve_tool_binary (PATH 优先 → extend-tools fallback)
    """
    if self.binary_path:
        return Path(self.binary_path).exists()
    from automisc.tools.paths import resolve_tool_binary
    return resolve_tool_binary(self.name) is not None
```

**改 3**：所有 adapter `self.binary_path or "toolname"` → 用 `resolve_tool_binary(self.name) or self.name`

```python
# 旧 (binwalk.py:87):
binwalk = self.binary_path or "binwalk"

# 新:
from automisc.tools.paths import resolve_tool_binary
binwalk = self.binary_path or resolve_tool_binary("binwalk") or "binwalk"
```

**为什么保留 fallback `"binwalk"`**：resolve_tool_binary 找不到时 subprocess 跑会自然报 FileNotFoundError，由 `_run_subprocess` 兜底返回 127

### 3.5 extend-tools/manifest.yaml 设计

```yaml
# extend-tools manifest — 工具版本 + 下载 URL + SHA256
# Owner 首次 clone 后跑 `pwsh ./extend-tools/install.ps1` 自动下 binary
# 不进 git 的二进制 (extend-tools/bin/win-x64/*) 通过这个文件追溯

        version: "12.87"
        url: "https://exiftool.org/exiftool-12.87_64.zip"
        sha256: "<占位>"
        # 注意: exiftool 实际是 exiftool(-k).exe + exe 文件, 解压后取 exiftool.exe
        post_extract:
          zip_contains: "exiftool(-k).exe"
          rename_to: "exiftool.exe"
      7zr:
        version: "24.07"
        url: "https://www.7-zip.org/a/7zr.exe"  # 独立版, 单文件
        sha256: "<占位>"
        # 独立版 7zr.exe 已经就是 binary, 不需要 rename
      foremost:
        version: "1.5.7"
        url: "https://github.com/raddyfiy/foremost/raw/master/binary/foremost.exe"
        sha256: "<占位>"
        # 第三方 fork, CTF 圈通用
  macos:
    # v0.5+ 暂空 (Homebrew 优先, 不强制 extend-tools)
    tools: {}
```

### 3.6 extend-tools/install.ps1 设计

**MVP** (per Owner "能用就行"，不追求完美):
- 读 manifest.yaml
- 对每个 tool:
  1. 检查 `<extend-tools/bin/win-x64>/<tool>.exe` 是否已存在
  2. 不存在 → 下载 URL 到 tmp
  3. exiftool 特殊处理：unzip + rename `exiftool(-k).exe` → `exiftool.exe`
  4. 移动到 `<extend-tools/bin/win-x64>/<tool>.exe`
  5. 计算 SHA256 并写回 manifest.yaml
- 不存在的 tool → 在 manifest.yaml 加 `available: false` 标记
- 完成 → 打印 "下次跑 `automisc-gui` 就能用了"

**失败处理**：单个工具下载失败不影响其他工具，最后汇总报告

**幂等性**：第二次跑直接跳过已下好的（按文件存在判定）

### 3.7 steghide / zsteg / stegseek 在 Windows unavailable

**改动小**：只在 docstring 加 "Windows 上不可用" 说明 + 内部 `check_available` 自动返回 False（因为找不到 binary）

**GUI 表现**：菜单 marker = ✗ + 点击执行报 stderr `"executable not found: steghide"`

**未来 v0.5+ 评估**（per Owner 决策）：
