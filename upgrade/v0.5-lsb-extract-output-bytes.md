## 1. 背景

### 1.1 问题陈述 (per Owner 22:55 反馈)

**实战链路**:
1. Owner 拖 `/Users/minzhizhou/Downloads/镜子里面的世界/steg.png` (v0.5-LSB-router 训练题, md5=f7e4cdc8...)
2. auto-run 跑 `lsb_detect` (per v0.5-lsb-detector + v0.5-train-011 修复后), 命中 1 SP:
   - `[5] lsb_file_header: lsb_detect 发现 lsb RGB row 存在可疑 png 文件: �PNG IHDR...IHDR...IDATx... (truncated, total 26934 bytes) {'file_type': 'png', 'magic_label': 'PNG image'}`
3. Owner 用 stegsolver.jar 确认: RGB 最低位隐写了一张 png 图片
4. Owner 点击 GUI 工具栏 "🎨 PNG LSB 智能提取" (menu_dock.py:106) → 调 `lsb_extract.py` (zsteg-based 老工具)
5. **bug 出现**: 输出 `/Users/minzhizhou/Downloads/LSB/flag11__lsb.txt` (乱码 + .txt 后缀)

**bug 根因** (`lsb_extract.py:232-239`):
```python
# v0.5-train-008 fix: text 通道 main 也写文件 (per Owner 决策)
# 命名规则: <stem>__lsb.txt (跟 file 分支一致, output_path_for 同目录)
main_extracted_path = output_path_for(
    file_path, suffix=".txt", purpose="lsb"  # ← bug 1: suffix 写死 .txt
)
main_extracted_path.write_text(
    main["text"], encoding="utf-8", errors="replace"  # ← bug 2: write_text 二进制当 text 写
)
```

**问题**:
- `suffix=".txt"` 写死 — 不管字节流是 PNG/zip/pyc/elf, 一律 .txt
- `write_text(...utf-8, errors="replace")` 把字节流当 UTF-8 text 解码 — 二进制字节 (e.g. PNG header `\x89PNG\r\n\x1a\n`) 解码出 `�PNG` 乱码 + `\r\n\x1a\n` 替换成 `?`
- 输出文件**无法**用 `file` 命令 / `binwalk` / 任何二进制工具处理

### 1.2 Owner 思路 (per 22:55 反馈原文)

> "**应该根据文件头输出对应的二进制文件**"
> "**比如 89 50 4E 47 是 PNG**"
> "**用 python 的 wb 来保存二进制文件内容**"

完全对齐 spec 设计:
- ✅ 根据文件头判断后缀 (89 50 4E 47 = .png, etc.)
- ✅ `write_bytes` 写二进制 (per Owner "wb")
- ✅ 复用 lsb_detect 25+ magic 库 (per lsb_detector spec §3.1)

## 2. 目标

### 2.1 能力 A: 修 lsb_extract.py 输出格式

**目标**: 修 `core/actions/lsb_extract.py` 写文件逻辑:
- **检测文件头** (前 N 字节 match known magic) → 决定后缀
- **写二进制** (`write_bytes`) 而非 `write_text`
- **fallback**: 没命中 magic → `.bin` 默认 (跟 lsb_bytes_extract 一致)

**判定逻辑** (复用 lsb_detect 已有):
```python
from automisc.core.actions.lsb_detect import _detect_file_header_hex

magic_hit = _detect_file_header_hex(raw_bytes)
if magic_hit:
    ext, label = magic_hit  # e.g. ("png", "PNG image")
    suffix = f".{ext}"
else:
    suffix = ".bin"  # fallback
```

**文件命名**:
- 旧: `flag11__lsb.txt` (乱码)
- 新: `flag11__lsb.png` (PNG 二进制) / `flag11__lsb.zip` / `flag11__lsb.bin`

### 3.2 核心算法 (line 232-239 重构)

```python
# v0.5-lsb-extract-output-bytes (per Owner 22:55 反馈):
# - suffix 写死 .txt → 改 magic 判定 (.png/.zip/.pyc/.elf/.bin)
# - write_text (UTF-8 decode) → write_bytes (二进制)
from automisc.core.actions.lsb_detect import _detect_file_header_hex

# 检测文件头决定后缀
def _decide_suffix(raw_bytes: bytes, default: str = ".bin") -> str:
    magic_hit = _detect_file_header_hex(raw_bytes)
    if magic_hit:
        ext, _ = magic_hit
        return f".{ext}"
    return default

# text 通道 main 也写文件 (per v0.5-train-008)
main_extracted_path = output_path_for(
    file_path, suffix=_decide_suffix(raw_bytes), purpose="lsb"
)
main_extracted_path.write_bytes(raw_bytes)  # write_bytes 替代 write_text
```

### 3.3 file 分支也修 (line 142)

```python
# 旧: out_path = output_path_for(file_path, suffix=hint_ext, purpose="lsb")
# 新: out_path = output_path_for(file_path, suffix=_decide_suffix(extracted, hint_ext), purpose="lsb")
```

实际上 line 142 已经传 `hint_ext` (e.g. ".zip" / ".png" from zsteg 解析), 改 _decide_suffix 让**实际 magic**覆盖 hint_ext (更准确)。

### 3.4 复用现有能力

| 复用 | 来源 | 用途 |
|---|---|---|
| `_detect_file_header_hex` | `core/actions/lsb_detect.py:202-220` (v0.5-lsb-detector 已实现) | 后缀判定 (25+ magic 库: ZIP/RAR/PNG/PYC 2/3/ELF/JPG/GIF/PDF/7Z/BZ2/GZ 等) |
