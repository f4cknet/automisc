**Ook! 语言**: BrainFuck 的变种, 用 "Ook." "Ook!" "Ook?" 三个 token 两两配对, 替代 BF 8 指令.

**automisc 现状**: `cipher_decoders.run_brainfuck` 只识别 `><+-.,[]` 8 字符, "Ook. Ook!" 等 token 被清理掉, 跑出空.

## 目标

新建 `run_ook` decoder:
1. 读 Ook! token (Ook. / Ook! / Ook?) → 配对 → 转 BF 字符
2. 调现有 `brainfuck_eval` 解释 BF
3. GUI 自动渲染到 "🔤 解密工具1" 分类 (跟 BrainFuck 同一组)
4. 测试 + 实战验证 (2 个 owner 文件: NUL 用 BF, Ook 文件用 Ook)

## 设计

**Ook → BF token 映射** (标准):
```
Ook. Ook. → +
Ook! Ook! → -
Ook! Ook? → >
Ook? Ook! → <
Ook. Ook! → .
Ook? Ook. → ,
Ook. Ook? → [
Ook? Ook. → ]
```

**实施路径**:
1. tokenize: 正则 `\bOok[.!?]` 抽 token (允许 Ook. / Ook! / Ook? 任意空格分隔)
2. 两两配对 → BF 字符 (8 种映射)
3. 不匹配 token 对 (单数 token / 非 Ook token) → 跳过 (宽容)
4. 调 `classical_ext.brainfuck_eval(bf_code)` 解释
5. 返回 `DecodeResult` (跟 cipher_decoders.py 其他 runner 同结构)

**复用 `brainfuck_eval`**: 不重新实现解释器, 只 token 转换 + delegate. Ook! 是 BF 变种, 语义同.
