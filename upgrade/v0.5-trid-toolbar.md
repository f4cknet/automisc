## 1. 背景

Owner 在 `extend-tools/bin/win-x64/TrID/` 下手动放了：
- `trid.exe` (108KB, TrID/32 v2.24, M.Pontello 2003-16)
- `triddefs.trd` (5.8MB, 13783 signatures, 完全本地 — 不需在线拉 defs)

TrID 是**基于 signature pattern** 识别的文件类型判定工具，跟 `file` (libmagic) 的差别：
- **`file`**：基于 magic bytes（前 4-8 字节固定模式），识别"是什么"
- **trid**：基于 data-byte pattern frequency（统计签名），识别"像什么"，**附带 probability 候选列表 + 推荐扩展名**

CTF 实战里 trid 跟 file 互补：
- file 给一行定性结论（"PNG image data, 800x600"）
- trid 给 N 个候选 + 概率（"80% ZIP / 10% DOCX / 5% EPUB"），能识别出 `伪装后缀` 类迷题（file 误判 ZIP 伪装 → trid 给 ZIP 90%+ 候选）

Owner 决策：GUI 工具栏新增入口"文件类型识别"，分类"共享基础工具"，**跟** `file` / `strings` / `binwalk` / `foremost` / `exiftool` / `xxd` **并列**。

## 4. 设计

### 4.1 adapter 接口（per file.py 模板）

```python
@register_tool
class TridAdapter(ToolAdapter):
    name = "trid"
    category = "shared"
    description = "基于 signature pattern 识别文件类型（候选列表 + 概率）"

    def run(self, file_path: str) -> ToolResult:
        # 1) 解析 binary: resolve_tool_binary("trid") — 命中 TrID/trid.exe
        # 2) 解析 defs: 跟 binary 同目录下的 triddefs.trd
        # 3) 调: trid.exe -d:<defs_path> -n:5 <file>
        # 4) 解析 candidate 行: "<pct>% (<.EXT>) <description> (<hex_id>/<count>/<something>)"
        # 5) 写 SuspiciousPoint:
        #    - top candidate (最高 percentage) → category="file_type_trid", severity=1
        #    - 扩展名伪装? (top candidate 跟文件后缀不一致) → severity=4 ⚠️ 高亮
```

### 4.2 输出格式（per 实测 2026-07-01 22:00）

输入：pyproject.toml (ASCII INI)
```
TrID/32 - File Identifier v2.24 - (C) 2003-16 By M.Pontello
Definitions found:  13783
Analyzing...

Collecting data from file: D:\...\pyproject.toml
100.0% (.INI) Generic INI configuration (1000/1)
```

输入：automisc.png
```
TrID/32 - File Identifier v2.24 - (C) 2003-16 By M.Pontello
Definitions found:  13783
Analyzing...

Collecting data from file: D:\...\automisc.png
100.0% (.PNG) Portable Network Graphics (16000/1)
```

输入：trid.exe 自身（自识别）
```
TrID/32 - File Identifier v2.24 - (C) 2003-16 By M.Pontello
Definitions found:  13783
Analyzing...

Collecting data from file: D:\...\trid.exe
 90.5% (.EXE) FreeBASIC 1.0x Win32 Executable (792408/92/73)
  3.5% (.EXE) Win32 Executable MS Visual C++ (generic) (31206/45/13)
  1.8% (.EXE) Microsoft Visual C++ compiled executable (generic) (16529/12/5)
  1.2% (.EXE) Win64 Executable (generic) (10525/10/4)
  0.8% (.EXE) Win16 NE executable (generic) (7038/12/2)
```

**候选行正则**：`^\s*(?P<pct>\d+(?:\.\d+)?)%\s+\(\.(?P<ext>[A-Z0-9]+)\)\s+(?P<desc>.+?)\s*\((?P<hex_id>[0-9A-Fa-f]+)/(?P<count>\d+)/(?P<rest>\d+)\)\s*$`

### 4.3 SuspiciousPoint 策略（per file.py 类比）

| 场景 | severity | category | suggested_action |
|---|---|---|---|
| trid 报 0 命中 | 1 | `no_candidates` | "trid 未识别（可能非常冷门格式或二进制损坏），看 xxd 头 16 字节对照 KNOWN_MAGIC" |
| trid top candidate **跟文件后缀一致** | 1 | `file_type_trid` | "trid 判定 <EXT> <desc>，正常" |
| trid top candidate **跟文件后缀不一致** | **4** ⚠️ | `file_type_mismatch` | "**文件后缀是 .<X> 但 trid 判定 <EXT> (<desc>)，疑似扩展名伪装**—— 改后缀再重跑确认" |
| trid top candidate 概率 < 50% | 3 | `file_type_low_confidence` | "trid 识别最可能 <EXT> 但概率仅 <pct>%，建议看 file 命令辅助判定" |

**sev=4 是亮点** — 实战里"后缀 vs trid 不一致"是 CTF 雕文件伪装的最常见套路之一（per `Challenge/QR_code.png` 实战 foremost 雕出 QR_code.png 但本身其实是 zip 这一类）。

### 4.4 完全离线约束（关键）

- trid **默认**会从 `mark0.net/TrID/defs/triddefs.zip` 在线拉 defs
- 违反 automisc 产品定位（per STRUCTURE.md §1 "完全离线"）
- **修法**：adapter 调 trid 时**显式** `-d:<absolute_path_to_triddefs.trd>` 强制本地 defs
- 绝对路径推导：`trid_dir = Path(trid_binary).parent`，`defs = trid_dir / "triddefs.trd"`
- 实测：owner 现成 trid + defs 同目录 → 路径直接 `TrID/triddefs.trd`，不需要新机制

### 4.5 paths.py 异名 subdir fallback 验证

per `tools/paths.py:resolve_tool_binary` 第 96-110 行（**异名 subdir fallback**）：
```python
for sub in bindir.iterdir():
    if not sub.is_dir() or sub.name.startswith("."):
        continue
    sub_other = sub / f"{name}{exe_suffix()}"
    if sub_other.exists():
        return str(sub_other)
```

`TrID/trid.exe` 命中 → `resolve_tool_binary("trid")` returns `...extend-tools/bin/win-x64/TrID/trid.exe` ✓
