## 1. 3 个 Bug + 修复

### Bug 1: 工具栏没有 base64 → 图片工具

**现象**：左侧工具栏 (TOOL_CATEGORIES) 22 adapter + 4 快捷 action，没 decoder。
**修复**：新增分类 `🔓 解码工具 (v0.5+ Decoders)` 含 `decoder:base64-image`。
**调用链**：菜单点击 → menu_dock callback 收到 `(name='base64-image', kind='decoder')` → main_window._on_dock_item_selected dispatch → `_run_decoder(name)` → DecodeRunner QThread。

### Bug 2: 工具栏没有 hex → ascii 工具

**现象**：同上，缺进制转换入口。
**修复**：
- 新增 `core/decoders/base_convert.py` — 自动探测 4 种格式（hex/binary/base64/base32）→ ASCII
- 优先级：binary > hex > base64 > base32（避免 binary 串 0101.. 误判 hex）
- 阈: `len(s) >= 2 + len % 2 == 0` for hex, `len(s) >= 8 + len % 8 == 0` for binary
- 新增分类 `🔢 进制转换 (v0.5+ Convert)` 含 `decoder:hex-ascii`
- CLI `automisc decode hex-ascii --file <txt>` 自动生成

**真题实测** (meihuai.jpg):
```bash
$ echo "28372c37290a" > /tmp/hex.txt
$ automisc decode hex-ascii --file /tmp/hex.txt
=== 🔢 Hex → ASCII ===
input: 28372c37290a
detected_format: hex
output_text: (7,7)
```
→ 这正是 meihuai.jpg 隐藏 QR 坐标的 hex 编码格式！

### Bug 3: LSB 抽到 secret/key 关键字没高亮

**现象**：`Challenge/steg.png` LSB 抽到 `"...the secret key is: st3g0_saurus_wr3cks"`，output 只高亮 flag_candidate（首段），整段 LSB text 没显示，**关键字 secret/key 不突出**。
**修复**：
- `output_view.append_lsb_text(lsb_text, channel)` 新方法:
  - 整段用**深黄底** + 浅黄字
  - 敏感词 (key/flag/secret/ctf/password) 额外用**深红底 + 黄字 + 粗体下划线**
  - 标题 `[LSB text] channel=b1,rgb,lsb,xy, len=117` 金黄粗体
- main_window._on_chain_finished 调 `append_lsb_text` 渲染整段

**端到端实测** (steg.png 走 lsb chain):

## 2. 关键设计

### 2.1 callback 签名升级

**旧**：`on_tool_selected(name)` — 单一 dispatch
**新**：`on_tool_selected(name, kind)` — kind ∈ {adapter, action, decoder}

owner 决定的 kind 区分:
- **adapter**: 22 个 core.adapter (subprocess + parse)
- **action**: 4 快捷 action (lsb_extract / fix_pseudo_zip / bruteforce_zip / bruteforce_rar)
- **decoder**: v0.5+ decoder (base64-image / hex-ascii)

main_window 单一入口 `_on_dock_item_selected(name, kind)`:
```python
if kind == "decoder":
    self._run_decoder(name)  # DecodeRunner (QThread)
else:
    self._run_tool(name)  # ToolRunner (QThread) - adapter + action 都走
```

### 2.2 base_convert 优先级

```
binary 优先 (chars 集最窄, 0/1 only)
  → hex (chars 集次窄, 0-9a-f)
  → base32 (不含小写)
  → base64 (chars 集最宽, 兜底)
```

binary 优先避免 `010101001110...` 串被判 hex。

### 2.3 工具栏名字前缀 `decoder:`

`decoder:base64-image` / `decoder:hex-ascii` —— callback 看到 `decoder:` 前缀就 dispatch 到 `_run_decoder`，否则 `_run_tool`。
