## 需求

拖入文件 → binwalk 检测到 embedded archive（zip / 7z / gzip）→ **自动**：
1. 调 binwalk -e 分离
2. 对分离出的 zip 跑**智能链**：
   - ① 直接解压（无密码）
   - ② 伪加密 → 修 flag bit 0 → 重试解压
   - ③ 真加密 → 4-6 位字典爆破

## 架构

### 1. `core/dag.py` — DAG 编排核心

- `Action` 抽象基类：`run(context) -> ActionResult`
- `DAGNode`：`Action` + `on_success` + `on_failure` 边
- `DAG.execute(context)`：拓扑串行执行 + `max_steps` 防死循环
- 上下文累积：每步 data 注入 `context[f"__step_{n}_{name}__"]` + `__log__`

### 2. `core/actions/binwalk_extract.py` — binwalk -e

- `BinwalkExtractAction.run(file_path, extract_dir)` → 调 `binwalk -e -C <dir>`
- 输出 `actual_extract_dir` + `extracted_files` 列表
- 失败：binwalk 不在 PATH / 0 提取 / timeout

### 3. `core/actions/zip_chain.py` — 三个 zip action

- `TryUnzipAction`：无密码解压，失败抛 `encrypted`
- `FixPseudoEncryptionAction`：清 `flag_bits bit 0`（LFH + CDH）→ zipfile 重解压
- `BruteforceZipAction`：4-6 位字典（数字 10^4-10^6 + 字母 4 位 ≈ 8.4M），用 `zipfile.read(pwd=...)` 试
- 检测伪加密算法：读 raw bytes，**加密位 set + 无 12 字节 PKCS#5 header** = 伪加密

### 4. `core/chains.py` — 预定义 DAG 模板

- `build_zip_chain_dag()`：try_unzip → fix_pseudo（v0.1 简化，不含 bruteforce）
- `build_zip_chain_with_bruteforce()`：try_unzip → fix_pseudo → bruteforce（v0.5+ 启用）
- `build_binwalk_extract_dag()`：binwalk_extract
- `find_embedded_archives(binwalk_stdout)`：检测输出含 "ZIP archive" / "7z archive" / "gzip"

### 5. GUI 集成 (`gui/main_window.py`)

- `_on_auto_chain_finished` 检查 binwalk 跑过 → 调 `_maybe_trigger_zip_chain_from_binwalk`
- `_maybe_trigger_zip_chain_from_binwalk`：
  1. 跑 `binwalk` 拿 stdout
  2. `find_embedded_archives` 检测
  3. 调 `BinwalkExtractAction` 分离
  4. 找 `.zip` 文件 → 对每个跑 `build_zip_chain_dag`
  5. output 区累积 chain log

## 关键技术决策

1. **DAG 框架而非简单 if/else 链**：
   - 节点化、可扩展、失败转移明确
   - 后续加 bruteforce / john zip2john 不动框架
2. **伪加密检测算法**：
   - 不能信 `7z`（它跟 python zipfile 一样只看 flag bit）
   - 不能信 `python zipfile.read(pwd=...)`（它强制要求密码）
   - **直接读 raw bytes** 检查 entry 加密位 + 12 字节 header 是否存在
3. **bruteforce 字典大小**：
   - 数字 4-6 + 字母 4 位 = 8.4M（v0.1 简化）
   - 不含字母 5/6 + 字母+数字 5/6（v0.5+ 改 john zip2john）
4. **chain 触发时机**：
   - 在 `_on_auto_chain_finished` 后（auto-run 跑完）→ 检查 binwalk 输出
   - 避免在 `dropEvent` 触发（dropEvent 还没跑 binwalk）
5. **失败转移简化**：
   - `fix_pseudo.on_success = None`（不重试 try_unzip，避免循环）
   - v0.5+ 优化：fix 后再试 try_unzip，验证后跑 bruteforce
