## 2. 修复

### 2.1 抽 helper `core/utils/output_path.py`

```python
def output_path_for(input_path, suffix=".png", purpose="base64"):
    """持续输出: <input_stem>__<purpose>.<suffix> 与 input 同目录"""
    # /Challenge/KEY.exe + (.png, base64) -> /Challenge/KEY__base64.png

def temp_path_for(input_path, suffix=".hash", purpose="rar_hash"):
    """临时辅助: <input_stem>.automisc_<purpose>.<suffix> 与 input 同目录, 跑完删"""
    # /Challenge/x.rar + (.hash, rar_hash) -> /Challenge/x.automisc_rar_hash.hash

def extract_dir_for(input_path, purpose="foremost"):
    """提取目录: <input_stem>__<purpose>/ 与 input 同目录"""
    # /Challenge/KEY.exe + foremost -> /Challenge/KEY__foremost/

def is_in_tmp(path):
    """判断 path 是否在系统 tmp 下 (resolved 路径比较, 兼容 /tmp 符号链接)"""
```

**命名规则**：
- 持续: `<stem>__<purpose>.<ext>`（用 `__` 双下划线分隔，避免和 input 重名）
- 临时: `<stem>.automisc_<purpose>.<ext>`（用 `.` 分隔，保持"看着像 input 的临时副本"语义）

### 2.3 `is_in_tmp` 设计

macOS `/tmp` 是符号链接 → `/private/tmp`，直接字符串检查会误匹配 `/Users/.../opencode/tmp/pytest-xxx` 这种项目根 tmp。改为**路径比较**：

```python
def is_in_tmp(path):
    """识别系统 tmp 目录:
    - /tmp (resolves to /private/tmp)
    - /private/tmp
    - /var/folders (macOS per-user tmp 前缀)
    """
    p = Path(path).resolve()
    for tmp in ["/tmp", "/private/tmp", "/var/folders"]:
        try:
            p.relative_to(Path(tmp).resolve())
            return True
        except ValueError:
            continue
    return False
```

`/Users/.../opencode/tmp/` 不识别（项目级 tmp 目录）—— pytest 自身用这个目录，不影响 helper 的语义判断。
