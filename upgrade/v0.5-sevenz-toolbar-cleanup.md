## 设计原则 (per Owner 20:03 拍板, 跨项目适用)

**GUI menu 显示策略**:

| 类别 | 是否显示 | 例子 |
|---|---|---|
| 操作类 (action) | ✅ 显示 | 解压 / 修复 / 爆破 / 解码 / 转换 |
| 探测类 (探测, auto_run 跑) | ❌ 可不显示 | foremost / binwalk / strings / file / exiftool / xxd / sevenz (探测) / zsteg / stegseek / tshark / tcpdump / grep / evtx_dump |

**理由**:
1. 探测类在 auto_run 已被自动跑, GUI 单独入口意义不大
2. GUI menu 越精简越好, 用户不被 "list vs extract" "探测 vs 解压" 混淆
3. 探测工具通常输出只是可疑点列表, 用户主动用 GUI 跑没额外价值
4. 解压 / 修复 / 爆破类是 "操作", 需要用户主动触发, 才有 GUI 入口价值
