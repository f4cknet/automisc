## 变更动机

- **binwalk -e 在 macOS 不可靠**（需要 `dd` 工具，silent 失败）— v0.5-DAG-chain 第一次实现时已发现 + 用 foremost 兜底
- **Owner 建议**：**统一引用 foremost**——binwalk 只做 detection，foremost 负责实际提取
- **可复用性**：foremost 提取能力应该独立 module（不绑 binwalk），其他场景（foremost 单独跑 / GUI 集成）也能直接用

---

## 设计

### 之前 (v0.5-DAG-chain 第一次)

```
core/actions/binwalk_extract.py
  - BinwalkExtractAction.run(): 调 binwalk -e 提取 (失败)
  + 直接调 foremost 兜底 (混在一个 class 里)
```

**问题**：binwalk -e 失败 fallback 到 foremost 的逻辑不清晰；foremost 代码不可独立复用

### 现在 (v0.5-DAG-chain-refactor)

```
core/actions/foremost_extract.py     ← 新增 (v0.5 refactor 核心)
  - ForemostExtractAction: 纯粹 foremost -t all 提取
  - find_foremost_extract(): helper 函数扫描输出

core/actions/binwalk_extract.py     ← 简化为 orchestration
  - BinwalkExtractAction: binwalk 检测 + delegate 给 ForemostExtractAction
```

**好处**：
1. **解耦** — 检测 vs 提取独立
2. **可复用** — `foremost_extract.py` 任何场景能用
3. **可测** — `ForemostExtractAction` 单独测，`BinwalkExtractAction` 测整合
4. **可替换** — v0.5+ 换 binwalk 检测为 zsteg 检测 / foremost 换 binwalk -e (在 Linux) 都容易
