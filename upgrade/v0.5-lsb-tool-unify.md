### 1.2 暴露的核心问题（per v0.5-train-011/012 实战）

- **v0.5-train-011**：lsb_detect 修复 12 组合改回 HWC interleaved（跟 zsteg `b1,rgb,lsb,xy` 一致）
- **v0.5-train-012**：lsb_extract 工具栏写真**117 byte** text 字节流；lsb_detect 检测命中的 SP 字节流是 **26934 byte PNG**——**两条路径字节流不同源**！根因：lsb_extract 调 `zsteg -e <channel>` 抽 stdout 命中的 channel，lsb_detect 用 PIL/numpy HWC 抽字节流，**没有公共函数**
- **v0.5-lsb-extract-pil-same-source**：spec 已起草但**未实施**

### 1.3 zsteg 在 Windows 上的现实

per `extend-tools/manifest.yaml` `unavailable.zsteg`：
> "Ruby gem, Windows 上需装 RubyInstaller (~300MB)"

可选装（RubyInstaller.org 5 分钟装好），但项目有意**不**装，原因：
1. 300MB 磁盘 for 1 tool
2. lsb_detect 已覆盖 zsteg 80% 能力 + entropy 增量
3. ruby-runtime + DevKit 引入系统级复杂度

### 1.4 Owner 决策（per 2026-06-28 10:36 输入）

- **合并方向**：lsb_detect + lsb_extract → lsb_tool（**实际还要加 lsb_bytes_extract = 3 合 1**）
- **auto-run 走 lsb_tool**：替代 lsb_detect
- **能力 ≈ zsteg**：单通道 + RGB 三通道 + 4 参数（channels/bit/scan_order/byte_bit_order）
- **保留 zsteg 短板补齐**：entropy + unique count + 跨通道异常（lsb_detect 独有）
- **保留单通道可疑点检测**：zsteg 在 Win 上做不到全通道覆盖

## 3. 设计

### 3.1 LSBToolAction 单一入口（3 mode × 4 参数 × 多 preset）

```python
# src/automisc/core/actions/lsb_tool.py
class LSBToolAction(Action):
    """LSB 隐写检测 + 字节流提取统一入口 (per v0.5-lsb-tool-unify)
    
    替代 3 个老 action (Phase 6 删除):
    - LSBDetectAction (readonly 探测, 12 组合 + entropy + magic)
    - LSBExtractAction (zsteg subprocess 抽 text/file)
    - LSBBytesExtractAction (chain lsb-bytes 4 参数)
    
    单一 PIL/numpy 字节流提取 + 4 参数化 + 3 mode.
    """
    name = "lsb_tool"
    
    def __init__(
        self,
        *,
        # 4 参数 (zsteg 兼容性)
        channels: str = "rgb",        # "r"/"g"/"b"/"a"/"rg"/"rb"/"gb"/"rgb"/"rgba"/"r3g2b3"
        bit: int = 0,                  # 0-7 (单 bit 位)
        scan_order: str = "row",       # "row" (xy) / "col" (yx)
        byte_bit_order: str = "msb",   # "msb" / "lsb" 字节内 bit 顺序
        # 3 mode
        mode: str = "detect",          # "detect" / "extract" / "extract_bytes"
        # preset
        preset: str | None = None,     # None / "all" / "np"
        # text/magic 阈值
        text_min_len: int = 20,        # printable ASCII 连续段最小长度 (per v0.5-train-011)
        entropy_threshold: float = 5.0,  # per v0.5-train-010 §2.2 MVP
        unique_threshold: int = 200,     # unique count ≥ 此值触发 entropy 异常
    ):
        ...
```

### 3.2 4 参数详解

| 参数 | 取值 | 含义 | zsteg 对应 |
|---|---|---|---|
| **channels** | `r`/`g`/`b`/`a`/`rg`/`rb`/`gb`/`rgb`/`rgba` + 复合串 | 像素通道组合 | `-c, --channels` |
| **bit** | 0-7 | 取该 bit 位（0=LSB, 7=MSB）| `-b, --bits N` |
| **scan_order** | `row` (= xy) / `col` (= yx) | 像素遍历顺序 | `-o, --order xy/yx` |
| **byte_bit_order** | `msb` / `lsb` | 字节内 bit 顺序 | `--lsb` / `--msb` |

**zsteg 不实现的功能**（per AGENTS §5.2 v1.x 评估）：
- `--shift N` / `--step N` / `--invert` / `--pixel-align`
- `-P, --prime` prime bytes/pixels
- `-s first/all/longest/none` 字符串查找模式（MVP 走默认 first）

### 3.3 3 mode 详解

#### mode="detect"（auto-run + GUI 探测按钮用）

- **输入**：图片路径
- **行为**：
  - 按 preset 跑组合（默认 12 组合，preset="np" 跑 G channel bit 0 col MSB）
  - 每个组合抽 byte stream
  - 3 维检测：
    1. **printable text**：连续 ≥20 字节 printable ASCII → sev=5
    2. **file magic**：50+ magic 库 + `file` 兜底 → sev=5
    3. **entropy anomaly**：Shannon entropy > 5.0 + unique count ≥ 200/256 + 跨通道差 ≥ 1.5σ → sev=4
  - 返回 SP 列表
- **写文件**：❌ readonly（per AGENTS §1 铁律 7）

#### mode="extract"（GUI 工具栏 "PNG LSB 智能提取"）

- **输入**：图片路径 + channels/bit/scan_order/byte_bit_order（默认 preset="all" 跑全组合）
- **行为**：
  - 跑组合（同 detect）
  - 命中 text 或 magic 的组合 → 抽 raw bytes
  - **写文件**：✅ 写到 samedir（`output_path_for` 同目录逻辑 + magic 决定后缀 .png/.zip/.bin）
- **替代**：原 `LSBExtractAction` zsteg subprocess 路径（Win 坏）→ PIL/numpy 字节流（per v0.5-train-012 修复方向）

#### mode="extract_bytes"（chain `lsb-bytes` + GUI 4 参数 dialog）

- **输入**：图片路径 + 4 参数 + chain 上下文
- **行为**：
  - 单组合（不走 preset）
  - 抽 byte stream → magic 检测 → 写文件
- **替代**：原 `LSBBytesExtractAction`
- **backward compat**：chain 名 `lsb-bytes` + GUI dialog 名 `lsb_bytes_dialog` 保留（仅内部实现替换）

### 3.4 preset 详解

| preset | 跑哪些组合 | 适用场景 |
|---|---|---|
| `None`（默认）| 单组合（按 4 参数）| chain lsb-bytes / GUI 4 参数 dialog |
| `"all"` | 12 组合（RGB 6 perm × row/col）+ 8 通道 entropy (per lsb_detect) | auto-run detect + GUI 工具栏智能提取 |
| `"np"` | G channel bit 0 col MSB（per v0.5-train-009 N=NP 默认）| N=NP 模式实战 |

### 3.5 字节流提取公共函数（v0.5-train-012 修复方向）

```python
def _extract_lsb_byte_stream(
    image: Image.Image,
    *,
    channels: str,
    bit: int,
    scan_order: str,
    byte_bit_order: str,
) -> bytes:
    """单组合字节流提取 (HWC interleaved, per v0.5-train-011 + v0.5-train-012)
    
    公共函数, 3 mode 复用, 字节流同源.
    """
    arr = np.array(image)  # HWC uint8
    if arr.ndim == 2:  # 灰度
        arr = arr[..., None]
    h, w, c = arr.shape
    
    # 1. channels 过滤
    channel_idx = _parse_channels(channels, c)  # e.g. "rg" -> [0, 1]
    
    # 2. scan_order 决定遍历顺序
    if scan_order == "row":
        pixels = arr.reshape(-1, c)  # row-major
    else:  # "col"
        pixels = arr.transpose(1, 0, 2).reshape(-1, c)  # col-major
    
    # 3. 提取 channel + bit
    bits = (pixels[:, channel_idx] >> bit) & 1  # shape (N_pixels, N_channels)
    
    # 4. 拼 byte (per byte_bit_order)
    flat = bits.flatten()  # channel interleave per pixel
    if byte_bit_order == "msb":
        bytes_list = []
        for i in range(0, len(flat) - 7, 8):
            byte = 0
            for j in range(8):
                byte = (byte << 1) | flat[i + j]
            bytes_list.append(byte)
    else:  # "lsb"
        bytes_list = []
        for i in range(0, len(flat) - 7, 8):
            byte = 0
            for j in range(8):
                byte |= flat[i + j] << j
            bytes_list.append(byte)
    
    return bytes(bytes_list)
```

**所有 mode 共用此函数**，保证 3 mode 字节流同源（修复 v0.5-train-012 bug）。

### 3.6 检测算法（3 mode 共享）

```python
def _detect_suspicious(byte_stream: bytes, channels: str, ...) -> list[SuspiciousPoint]:
    """3 维检测 (per lsb_detect 已实现)"""
    sps = []
    
    # 1. text 判定 (per v0.5-train-011 修复: ≥20 字节连续 printable)
    text = _extract_printable_text(byte_stream, min_len=text_min_len)
    if text:
        sps.append(SuspiciousPoint(severity=5, category="lsb_text", ...))
    
    # 2. magic 判定 (per lsb_detect._detect_file_header_hex 50+ magic 库)
    magic = _detect_file_header_hex(byte_stream)
    if magic:
        sps.append(SuspiciousPoint(severity=5, category="lsb_file", ...))
    
    # 3. entropy + unique (per v0.5-train-010 §2.2)
    entropy = _shannon_entropy(byte_stream)
    unique = _unique_count(byte_stream)
    if entropy > entropy_threshold and unique >= unique_threshold:
        sps.append(SuspiciousPoint(severity=4, category="lsb_entropy", ...))
    
    return sps
```

        )
