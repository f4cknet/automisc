## 1. 背景

### 1.1 v0.5-train-018 实战回顾

Owner 2026-06-29 22:55 实战 `flag.vmdk` (3MB VMware4 disk image):
- 7z 26.02 不支持 vmdk (per `v0.5-sevenz-extract.md` doc-fix)
- foremost 1.5.7 Win length overflow bug 雕不出 vmdk (audit.txt 显示 "Length: 8EB")
- 实战答案: `qemu-img convert -f vmdk -O raw flag.vmdk flag.raw` → foremost 雕 raw → 2 flag 文件 (NULL brainfuck + flag_part_two.txt Ook!)

### 1.2 qemu-img Win 装法选项

| 方案 | 描述 | 优劣 |
|---|---|---|
| **A. qemu setup.exe 全装** | 装 `qemu-w64-setup-2025xxxx.exe` → 默认 `C:\Program Files\qemu\qemu-img.exe` | 500MB+ 全装, 包含 GUI qemu-system; PATH 不一定注册 |
| **B. qemu-img standalone zip** | 找 `qemu-img-win-x64-static.zip` (只含 qemu-img + 依赖 DLL) | ~20MB, extend-tools/ 装, resolve_tool_binary 找 |
| **C. 系统 PATH 装** (不 extend-tools) | 装 setup.exe + 手动加 `C:\Program Files\qemu\` 到 PATH | 不动 extend-tools, paths.py 通过 shutil.which 自动找 |

**Owner 22:58 拍板"装"** = 装到 extend-tools/ (B 方案优先). 本 spec 实施方案 B.

### 1.3 qemu-img Win standalone prebuilt 来源

- **qemu-utils Win prebuilt** (Stefan Weil 维护): `https://qemu.weilnetz.de/w64/` 提供整套 qemu Win prebuilt, 但**全 setup**, 不单 qemu-img
- **Chocolatey qemu-img package**: 需 choco, 不在 extend-tools 范围
- **GitHub qemu-img-portable** (community): 部分作者提供 standalone, 需找可靠源

**实际挑战**: qemu-img Win standalone 独立 zip **难找**, setup.exe 装法 = A 方案 (500MB).

**实用妥协** (本 spec 采用): **B 方案变体**:
- 装 `qemu-w64-setup-xxxx.exe` (setup.exe 静默装 `/S` flag, NSIS installer 走 standard install)
- setup 装完 `C:\Program Files\qemu\qemu-img.exe` (~10MB 单文件)
- install.ps1 Stage 1 复制 `qemu-img.exe` 到 `extend-tools/bin/win-x64/`
- 不装 qemu-system (500MB) — install.ps1 setup 用 `/COMPONENTS="qemu-img"` 或类似 flag 限制

> **注**: QEMU 官方 NSIS installer 支持 `/S` 静默装, 但**不一定**支持组件选择. 如果 `/COMPONENTS` 不支持, 退而求其次: 装全 setup (500MB), install.ps1 Stage 1 只复制 `qemu-img.exe` 到 extend-tools/ (其余 qemu-system tools 留在 `C:\Program Files\qemu\` 不动).

          install.ps1 Stage 1 调 setup.exe /S 静默装 + 复制 qemu-img.exe 到 extend-tools/ (10MB).

### 2.2 install.ps1 Stage 1 加 qemu-img entry (setup.exe 静默装, 不复制)

```powershell
@{
    name = "qemu-img"
    version = "9.1.0"  # TODO: 装时定
    url = "https://qemu.weilnetz.de/w64/qemu-w64-setup-2025.05.12.exe"  # TODO: URL confirm
    target = $null  # 不复制, 装到 C:\Program Files\qemu\, paths.py 走 PATH 找
    post_extract = "qemu_img_setup_silent"
}
```

加 `qemu_img_setup_silent` post_extract handler (静默装, 不复制, 0 extend-tools 占用):

```powershell
"qemu_img_setup_silent" {
    # 1. 静默装 QEMU NSIS installer (装到 C:\Program Files\qemu\)
    $qemu_install_path = "C:\Program Files\qemu"
    $installer = $tmp  # 已下载的 setup.exe
    Write-Host "[install] QEMU NSIS silent install to $qemu_install_path ..."
    $proc = Start-Process -FilePath $installer -ArgumentList "/S" -Wait -NoNewWindow
    if ($proc.ExitCode -ne 0) {
        throw "QEMU setup.exe silent install failed, exit code: $($proc.ExitCode)"
    }
    if (-not (Test-Path "$qemu_install_path\qemu-img.exe")) {
        throw "QEMU installed but qemu-img.exe not found at $qemu_install_path"
    }
    # 2. (不复制) qemu-img.exe 在 $qemu_install_path, paths.py 走 shutil.which 找
    #    如果 NSIS setup 没自动加 PATH, Owner 手工:
    #    [Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\qemu", "User")
    # 3. 清 setup.exe download
    Remove-Item $installer -Force -ErrorAction SilentlyContinue
    Write-Host " OK" -ForegroundColor Green
    Write-Host "[note] qemu-img in PATH (NSIS silent install), extend-tools/ 0 占用" -ForegroundColor Gray
}
```

**为什么 0 extend-tools 占用** (per v0.5-platform-extend-tools + v0.5-windows-only 治理):
- extend-tools/ 设计**只装单文件** (e.g. 7z.exe / file.exe / steghide.exe) 不复制大目录
- QEMU 全 setup 500MB+ 含 qemu-system GUI tools, 实战只用 qemu-img (10MB) — **不**复制 500MB
- 装到 `C:\Program Files\qemu\` 是 Windows 习惯, 跟系统 PATH 注册
- paths.py 走 shutil.which("qemu-img") 优先, 命中 `C:\Program Files\qemu\qemu-img.exe`
- extend-tools/bin/win-x64/ 装 steghide / 7z / file / foremost / vim92/xxd 这类**单文件**工具, 不动 qemu-img

### 2.3 paths.py 加 qemu-img (自动通过 resolve_tool_binary)

**不**动 paths.py — `resolve_tool_binary("qemu-img")` 已经支持:
1. `shutil.which("qemu-img")` (PATH 优先) — 装 setup.exe 注册 PATH 后命中
2. `extend-tools/bin/win-x64/qemu-img.exe` (fallback) — install.ps1 复制后命中

owner CLI 实测: `qemu-img convert -f vmdk -O raw flag.vmdk flag.raw`.

### 2.4 不新建 qemu-img adapter (per §5.2 实战 1 道不升架构)

实战 1 道同类 → **不**新建 adapter, owner CLI 调 `qemu-img` 即可:
- `qemu-img info <file>` 查格式
- `qemu-img convert -f <in_fmt> -O <out_fmt> <in> <out>` 转换
- 实战 ≥ 3 道同类再升架构 (新建 `qemu_img_adapter` 提供 GUI 入口)

**`auto_run` 不挂 qemu-img** (per AGENTS §1 铁律 7, auto_run 纯探测不操作, qemu-img convert 是写盘操作).

> **📝 决策反转 2026-06-29 23:25 (per Owner 23:23 拍 B 升架构, 反转本 spec §2.4 旧决策)**: 本 spec 写"不建 adapter"是按 §5.2 实战 1 道同类**不升架构**的"标准做法". 实战 1 道同类**仍**维持 §2.4 决策 (owner CLI 跑 qemu-img 即可), **但** owner 装完**期望** GUI 入口 = 实战需求变更. ## 4. 风险

| 风险 | 概率 | 缓解 |
|---|---|---|
| QEMU NSIS /S 静默装失败 | 中 | install.ps1 调 setup.exe /S, 失败抛 + warn continue (per v0.5-tool-install-batch-1 模式) |
| qemu-img.exe 单文件装到 500MB setup 里, 卸装全 setup 风险 | 中 | 暂不卸, install.ps1 只复制单文件, 其余 qemu-system 留在 C:\Program Files\qemu\ |
| qemu-img 装后依赖 glib-2.dll 等 runtime DLL | 中 | QEMU 装时 setup 装 glib runtime 到 C:\Program Files\qemu\ 同目录, 复制 qemu-img.exe 后**仍需要**这些 DLL — install.ps1 复制整个 `C:\Program Files\qemu\` 子目录 (qemu-img.exe + glib DLL + 各种 .dll) 到 extend-tools/, 跟 vim92/xxd 异名 subdir 模式一致 |
| URL 失效 (qemu.weilnetz.de 经常换 URL) | 中 | install.ps1 失败 → warn continue, Owner 手工装 + 加 PATH |
| vmdk 实战只 1 道, 升架构过早 | 高 | 不建 adapter, 实战 ≥ 3 道再升 (per AGENTS §5.2) |
