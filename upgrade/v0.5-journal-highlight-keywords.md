## 1. 背景

### 1.1 Bug 1：`append_suspicious` 没做关键字高亮

`src/automisc/gui/output_view.py:115-147` 的 `append_suspicious()`：
- 仅按 `SEVERITY_COLORS` 上色（5 红 / 4 橙 / 3 黄 / 2 绿 / 1 灰）
- **完全没**对 `matched_pattern` 里的 `secret` / `key` / `password` / `pass` 子串做高亮
- 唯一走关键字高亮的是 `append_lsb_text()`（line 198 用 `_SENSITIVE_KEYWORDS`）—— **只对 LSB 抽出文本生效**

**后果**：
- strings 报 "found password='XXX' in entry_name" → SP severity=2，绿色
- 但 GUI 看不到 password 字样在哪
- Owner 在 LSB 题已经发现：ctf/flag 高亮，secret/key/password 没高亮

### 1.2 Bug 2：真加密 zip 链继续 brutalforce

`src/automisc/core/actions/zip_chain.py:97-103` `TryUnzipAction` 已经识别真加密（RuntimeError → fail "zip is encrypted"），但 chain 仍 `on_failure → BruteforceZipAction` 跑 8.4M 字典。

`src/automisc/gui/main_window.py:285` 触发 `build_zip_chain_with_bruteforce()`，导致主线程同步跑 8.4M 字典（per v0.5-train-003 实测 100 密码 1.6s × 84000 = 37 小时）。

**owner 要求**：真加密 zip → chain 立刻停 + journal 记 "真加密, 密码未知"。

### 1.3 Bug 3：journal 缺密码线索归集

strings / exiftool / auto_run 跑出的"可能密码"散在各处，**没**汇总到 journal 让 Owner 一眼看到。

## 2. 设计决策（Owner 决策已含在原话中）

### Q1: 关键字高亮白名单 = ?

**Owner 答**：把 `secret` / `key` / `password` / `pass` 加入（CTF 常见密码线索词）。
- 已有白名单（`rule_scanner.py:39`）：`key`, `flag`, `secret`, `ctf`, `password` —— **缺 `pass`**
- 本次补：`pass` 加入（避免 "passphrase" 等长词被误高亮 → 用 `\bpass\b` regex 限定）

### Q2: 真加密 zip 立刻停 + 跳到哪个 step？

**Owner 答**：真加密 → chain 在 `try_unzip` 失败后**直接终止**（不 brutalforce）；`fix_pseudo` 仍然跑（伪加密也归真加密一类前先 fix）；bruteforce 改为**owner 显式触发**（CLI `--bruteforce-limit 100` 或 GUI 弹对话框）。

### Q3: journal 密码线索归集 = ?

**Owner 答**：在 auto-run 流程结束后，把所有 SP 里的"密码线索候选"汇总写 journal：
- 扫描所有 SP.matched_pattern 命中 `_SENSITIVE_KEYWORDS` 任意子串的
- 写到 journal add_event，kind="密码线索候选"，severity=0（信息级）

### 3.2 decision tree

```
TryUnzipAction.run(zip_path)
  ├─ 1) extractall(no password) 成功 → success, extracted_to
  ├─ 2) RuntimeError (encrypted) → fail, data["encrypted"]=True
  └─ 3) BadZipFile → fail, message="bad zip"

build_zip_chain_dag() (v0.5+ 改):
  try_unzip
    ├─ success → 链终止
    └─ fail
       ├─ data["encrypted"]=True (真加密) → 链终止 (per Owner)
       └─ 其他 (损坏/伪加密) → fix_pseudo_encryption
          ├─ success → 链终止
          └─ fail → 链终止 (失败记录)

build_zip_chain_with_bruteforce() (改名: build_zip_chain_bruteforce_on_demand):
  try_unzip → 失败 (含真加密/伪加密) → fix_pseudo_encryption → 失败 → bruteforce
  (Owner 显式 trigger 才会跑这条链, GUI 弹窗)

GUI _maybe_trigger_zip_chain_from_binwalk:
  检测到 zip → 弹 QMessageBox:
    "检测到 {N} 个 zip, 选模式?"
    [Yes - 普通链]  [Bruteforce - 慢]  [No - 跳过]
  默认: Yes (避免 owner 误触 brutalforce 卡 GUI)
```

def append_suspicious(self, sp: SuspiciousPoint) -> None:
    from automisc.core.utils.rule_scanner import _SENSITIVE_KEYWORDS
    # ... existing SEVERITY_COLORS + long hex 占位符 ...

    # 关键字高亮: 把 sp.matched_pattern 拆段, 命中 _SENSITIVE_KEYWORDS 的位
    # 用 [红底 + 加粗 + 下划线] 渲染
    pattern = sp.matched_pattern[:200]
    text_lower = pattern.lower()
    cursor.movePosition(QTextCursor.End)

    # 标题
    fmt_header = QTextCharFormat()
    fmt_header.setForeground(SEVERITY_COLORS.get(sp.severity, QColor("white")))
    fmt_header.setFontWeight(QFont.Bold)
    header = f"  [{sp.severity}] {sp.category}: "
    cursor.insertText(header, fmt_header)

    # 关键字高亮: 在 pattern 上扫 _SENSITIVE_KEYWORDS
    # 类似 append_lsb_text 的 highlight 逻辑, 但简化 (不需要分块)
    fmt_kw = QTextCharFormat()
    fmt_kw.setBackground(QColor(180, 0, 0))
    fmt_kw.setForeground(QColor(255, 255, 0))
    fmt_kw.setFontWeight(QFont.Bold)
    fmt_kw.setFontUnderline(True)

    fmt_normal = QTextCharFormat()
    fmt_normal.setForeground(SEVERITY_COLORS.get(sp.severity, QColor("white")))
