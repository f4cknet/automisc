### 3.2 新建 `core/decoders/cipher_decoders.py`

**12 个 cipher runner**（统一签名 `(text | None, file_path | None, **kwargs) -> DecodeResult`）：
- 复用 `core/encoders/classical.py`：`caesar`、`rail_fence`、`pigpen`
- 自编写 9 个：`morse`、`bacon`、`xxencode`、`uuencode`、`jsfuck`、`jjencode`、`quoted_printable`、`brainfuck`、`bubblebabble`
- **pigpen 重写**：CTF 给的猪圈符号是 unicode 字符（`⌜⌝⌞⌟`、`∴∵` 之类），不是简单字母映射。用经典的 **9 网格 + X + V 反转** 真实符号表做反向解码。
- **凯撒默认 shift=3**（per 经典定义），但接受 `--shift N` 参数覆盖
- **栅栏默认 rails=2**，但接受 `--rails N` 参数覆盖
- **培根**默认 A=a/B=b（24 字母版），接受 `--variant {24,26}` 参数
- **jsfuck/jjencode 纯 Python 解析**：
  - jjencode 解码：把 `=ABC...==` 这种模式识别出来，提取 `String.fromCharCode(...)` 调用（pyjsfuck 风格：递归 eval `[]["filter"]["constructor"]("return String.fromCharCode")()` + `split|sort|concat`）
  - jsfuck 解码：识别 `[]()!+` 六字符表达式，用 Python 模拟 JS 表达式求值（eval `[+!+[]+[+[]]+...]` 这种）

所有 runner 返回统一 `DecodeResult`，失败 `error=...` 不抛。

## 6. 风险

- **pigpen 重写**：CTF 题目给的猪圈符号格式不一（有些用 ⌜⌝⌞⌟，有些用 ∴∵∶），不保证全覆盖；runner 接受 `--variant {simple,grid,x,v}` 参数切换
- **jsfuck/jjencode 纯 Python 解析**：完整实现 JS 子集（`+ - ! []` 等价转换 + 字符串拼接）有边界情况；首次实现可能只能覆盖简单 payload（CTF 常见 `[]["filter"]["constructor"]("return ...")()` 模式），复杂 payload 用 error 提示用户 fallback
- **bubblebabble**：算法是固定的 5 元组 + 数字字符规则，纯 Python 实现稳定

## 9. v0.5-cipher-decoders-textfix (text_only 自动声明)

**Owner 反馈**（2026-06-14 19:14）：摩尔斯 input `../.-../---/...-/./-.--/---/..-/-.-.--/...../..---/-----` 输入 input 区，点击"摩尔斯解密"工具栏 → 提示 `[!] no file selected`。

**Root cause**: GUI `_run_decoder` 之前的 `text_based_decoders` 是**手写硬编码 set**（`{hex-ascii, base16, base32, ... rot47, base64-custom, base64-stego}`）。新加的 12 个 cipher + 2 占位没注册进去 → 走 file 模式 → 没 file 报错。

**修复**：
- `DecoderSpec` 加 `text_only: bool = False` 字段（向后兼容）
- 12 cipher + 14 base/rot + 2 占位 + hex-ascii 共 **28 个 decoder** 在注册时设 `text_only=True`
- GUI `_run_decoder` 改成读 `spec.text_only`（不再硬编码 list）
- 老 file-based decoder (`base64-image`, `coords-qr`) 仍 `text_only=False`，行为不变
- coords-qr 特殊 override（有 current_file 时走 file 模式）保留
