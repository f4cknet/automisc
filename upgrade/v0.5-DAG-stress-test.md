## 详细输出

### 01_pseudo.zip (zip chain)

```
[1] try_unzip                 FAIL   zip is encrypted: ...
[2] fix_pseudo_encryption     OK     fixed 2 flag_bits; unzipped to .../01_pseudo_unzipped
```

✅ 链终止于 step 2 (fix_pseudo OK)，flag 在 `01_pseudo_unzipped/flag.txt`。

### 02_real_4digit.zip (zip-full chain, 无 limit)

```
[1] try_unzip                 FAIL   zip is encrypted
[2] fix_pseudo_encryption     FAIL   fix failed, restored from backup
[3] bruteforce_zip            OK     FOUND password='7531' (tried 7532/8421616)
                                 unzipped to .../02_real_4digit_bruteforced
```

✅ 密码 `7531`，3.15s。**注意**：链的 step 2 失败后**回滚了 backup**（fix_pseudo 的安全机制）—— step 3 拿到的还是原始加密文件，bruteforce 成功。

**关键证据**：fix_pseudo 失败时**自动恢复**——避免了"fix 失败后变成损坏文件 + 不能 bruteforce"的灾难。

### 03_png_with_zip.png (foremost chain)

```
[1] foremost_extract          OK     foremost extracted 1 files
                                 /tmp/.../foremost_out/zip/00000000.zip
```

✅ 手动 unzip 00000000.zip → `flag{png_embedded_zip_003}`。

### 04_nested_pseudo.zip (zip chain)

```
[1] try_unzip                 OK     unzipped to .../04_nested_pseudo_unzipped (no password)
                                 inner.zip inside
```

⚠️ **内层未自动递归**。手动二次跑：
```bash
$ automisc chain --chain zip --file .../04_nested_pseudo_unzipped/inner.zip
[1] try_unzip                 FAIL
[2] fix_pseudo_encryption     OK
```

✅ 半自动完成。**这是 v0.5-DAG 已知局限**：DAG 是静态拓扑，**不会**对抽出的文件递归调用 chain（需要用户二次触发）。

### 05_multi_carve.png (foremost chain)

```
[1] foremost_extract          OK     foremost extracted 1 files
                                 /tmp/.../foremost_out/zip/00000000.zip
```

✅ foremost 提取了 zip（忽略 jpeg，因为 jpg 内容是 `\x00` 没意义）。手动 unzip → `flag{multi_carve_005}`。

### 06_png_only.png (foremost chain)

```
[1] foremost_extract          FAIL   foremost exit=0 but no files extracted
```

✅ **graceful 退化**：foremost 跑完没找到嵌入文件，chain 报告失败但**不挂**。符合"半自动化辅助工具"定位——告诉用户"foremost 没找到"而不是 crash。

## 已知局限

1. **DAG 不递归**（04 半自动）—— 抽出 inner.zip 后需用户手动二次触发 chain  
   - **v0.5+ 改进方向**：加 `recursive=True` 选项，foremost 链抽出后自动喂入 zip 链
2. **bruteforce 默认字典不含字母+数字混合 5-6 位**（需 `AUTOMISC_BRUTEFORCE_FULL=1`）  
   - Owner v0.5 决策：默认字典 = 数字 4-6 + 字母 4，避免一小时级别误启动
3. **foremost 1.5.7 提取 docx 需 `-t all`**（默认 docx 被认作 zip）  
   - v0.5-FIXED-001 修了 `file_types="all"` 默认
4. **套娃检测**（找 foremost 输出中的 .zip）`find_embedded_archives` 已实现，但**没自动接 zip_chain**（v0.5+ 接）
