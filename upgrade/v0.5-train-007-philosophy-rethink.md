## 2. automisc 真正哲学 (owner 2026-06-20 12:02 拍板)

**核心三原则**:
1. **工具 = 找可疑点** (探测)
2. **journal = 可疑点累积** (结果池)
3. **做题人 = 决策下一步** (人工判断, **不抢 flag**)

**实施意图**:
- `auto_run` 改名 → **`find_suspicious_from_<type>`** (type = picture / traffic / archive / binary)
- **每类独立工具池**, **不**互相串联
- **只探测, 不雕不修不爆**
- 结果**全**写 journal
- **不**触发 chain (chain 留给**人工**触发: `automisc chain --chain zip` / GUI 工具栏按钮)

## 4. Owner 真实意图的边界

**保留** (人工触发的 chain / 工具):
- `build_zip_chain_dag` / `build_zip_chain_with_bruteforce` — CLI `automisc chain --chain zip` / GUI 工具栏 fix_pseudo_zip 按钮
- `build_foremost_extract_dag` — 还原到**单 step** (foremost 雕, **不**加 dd fallback) — 人工触发 OK
- `build_lsb_extract_chain` — 还原
- 工具栏 foremost / binwalk 按钮 (人工触发雕) — 保留
- **binwalk adapter** — 探测模式 (`binwalk <file>` 不带 `-e`), 写 SP 到 journal — **保留**

**撤回** (自动化分离):
- 我刚做的 v0.5-binwalk-vs-foremost 改动 (5 文件 + dd_extract.py + 2 文档) — **全部 revert**
- `gui/main_window.py:_maybe_trigger_zip_chain_from_binwalk` — **删**
- `auto_run` 触发链的所有路径 — **删**

**新设计** (替代 auto_run):
- `find_suspicious_from_picture` (图片专用探测器)
  - 跑 zsteg / steghide / exiftool / binwalk (探测模式) / strings / file
  - **不**跑 foremost (雕文件) / steghide extract (抽数据)
  - 结果写 journal
- `find_suspicious_from_traffic` (流量包专用探测器)
  - 跑 pcap_protocol_router / tshark / strings / file
  - **不**跑链触发
- `find_suspicious_from_archive` (压缩包专用探测器)
  - 跑 sevenz / unzip (探测模式: `unzip -l` / `7z l`, 不解压) / john (爆破**不**在 auto_run, 仅在**工具栏**)
  - **不**触发 fix_pseudo
- `find_suspicious_from_binary` (二进制专用探测器)
  - 跑 file / strings / binwalk (探测模式) / exiftool

**关键边界**:
- **`auto_run` (rename) = 纯探测, 不雕不修不爆**
- **工具栏按钮 = 雕/修/爆, 人工触发**
- **`automisc chain --chain X` = 链, 人工触发**
