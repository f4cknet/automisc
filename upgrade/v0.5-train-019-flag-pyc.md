## 3. AI 兜底解题（uncompyle6 反编译 + 字节码交叉验证）

### 3.1 uncompyle6 反编译源码（v0.5-pyc-magic-sniffer 部署的解码器）

```python
import base64

def encode():
    flag = '*************'
    ciphertext = []
    for i in range(len(flag)):
        s = chr(i ^ ord(flag[i]))
        if i % 2 == 0:
            s = ord(s) + 10
        else:
            s = ord(s) - 10
        ciphertext.append(str(s))
    return ciphertext[::-1]


ciphertext = [
 3, 4, 5, 6, 7, 8, 9, 5, 10, 11, 12, 13, 14, 15, 16, 8, 17, 18,
 19, 20, 21, 22, 23, 15]
return
```

### 3.2 ❌ uncompyle6 反编译有 2 个 bug

**Bug A — 顶层 ciphertext 反编译错**:
- uncompyle6 输出 `[3, 4, 5, ..., 15]` 24 个数字
- 实际字节码 LOAD_CONST 顺序的真实字符串是 24 个: `['96', '65', '93', '123', '91', '97', '22', '93', '70', '102', '94', '132', '46', '112', '64', '97', '88', '80', '82', '137', '90', '109', '99', '112']`
- uncompyle6 把 co_consts **索引**当成值输出了（应该解析 consts[3]='96' 输出 '96', 它输出 3）

**Bug B — encode 函数内 `*************` 13 字符占位**:
- 实际 encode 函数内 co_consts[1] = `*************` (13 字符) — 这是 uncompyle6 原样输出
- 真实 flag 长度 = 24 (顶层 ciphertext 长度)
- uncompyle6 没有把 encode 内的 flag 字符串解析为真实长度（如果原 Py2.7 源码 flag 真的是 24 字符, uncompyle6 也只能显示占位符）

### 3.3 字节码交叉验证（per tools/_debug/disasm4.py）

xdis `Bytecode` 反汇编确认:
- encode 函数算法: `偶数 + 10, 奇数 - 10` (跟 uncompyle6 反编译**一致**)
- 顶层 LOAD_CONST 顺序 24 个字符串 = encode 输出顺序

### 3.4 反推 flag

```python
# 顶层 ciphertext = encode(flag) 输出 (encode 返回 ciphertext[::-1], 但题目给的 ciphertext 已经按 encode 输出顺序)
ciphertext = ['96', '65', '93', '123', '91', '97', '22', '93', '70', '102', '94', '132', '46', '112', '64', '97', '88', '80', '82', '137', '90', '109', '99', '112']

def encode_v1(flag):
    ciphertext = []
    for i in range(len(flag)):
        s = chr(i ^ ord(flag[i]))
        if i % 2 == 0:
            s = ord(s) + 10
        else:
            s = ord(s) - 10
        ciphertext.append(str(s))
    return ciphertext[::-1]

# 反推: 对 ciphertext[::-1] 按 i 顺序反推 flag (因为 encode 返回 ciphertext[::-1])
def reverse_v1(ct):
    flag_chars = []
    for i, val in enumerate(ct):
        s_int = int(val)
        if i % 2 == 0:
            orig = s_int - 10  # 反向 + 10
        else:
            orig = s_int + 10  # 反向 - 10
        s_char = chr(orig)
        flag_ord = ord(s_char) ^ i
        flag_chars.append(chr(flag_ord))
    return ''.join(flag_chars)

flag = reverse_v1(ciphertext[::-1])
# 验证
assert encode_v1(flag) == ciphertext
print(flag)  # flag{Y@e_Cl3veR_C1Ever!}
```

**答案**: `flag{Y@e_Cl3veR_C1Ever!}` (24 字符)

### 3.5 关键洞察

1. **encode() 返回 ciphertext[::-1]**: 这是个"输出倒序"操作, 容易混淆 — 解码时要把"题目给的 ciphertext"再 [::-1] 一遍回到 encode 内部顺序
2. **uncompyle6 反编译顶层 consts 列表有 bug**: 不能信任反编译源码里的 list 字面量数值, 一定要交叉验证字节码 LOAD_CONST 顺序
3. **<media>** 完整解题步骤沉淀在 `tools/_debug/solve_v4.py` (不入主分支, 仅一次性脚本)
