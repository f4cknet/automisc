# v0.5-decoder-arch-doc — 文档化 GUI 工具来源 (tools/ vs decoders/)

## 背景

per Owner 2026-06-20 20:38 拍板:

> 虽然在 gui 工具栏中虽然看到了有不少解密工具, 但是我刚刚大致看了下后端 tool,
> 好像后端没有真正的这些解密/解码工具, 比如这题的 brainfuck,
> 我并没在 src/automisc/tools/ 中看到有解密工具1对应的目录

Owner 看到 GUI 菜单 "解密工具1" (12 cipher: 凯撒/培根/栅栏/猪圈/摩尔斯/xxencode/uuencode/jsfuck/jjencode/BrainFuck/BubbleBabble) 但 `tools/` 下没对应目录, 误以为后端缺实现.

**真相**: GUI 工具栏有 **两种来源**, 不是所有 GUI 工具都在 `tools/`:
- `tools/` adapter — subprocess 调外部 CLI (file/strings/binwalk/sevenz/...)
- `core/decoders/` decoder — 纯 Python 计算 (caesar/rot/base64/bf 解释器/...)

cipher 解密是纯计算 (caesar shift / rot13 / base64.b64decode / BF 解释器), 放在 `core/decoders/cipher_decoders.py`, **不放** `tools/cipher/` (反分层, 空壳 adapter).
