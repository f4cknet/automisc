> **status**: 🔄 in-progress (2026-06-29 22:10 journal_clear 已推远端, 实施代码 + 单测中)

## 1. 背景

### 1.1 v0.5-platform-extend-tools 现状 (per `upgrade/v0.5-platform-extend-tools.md`)

`tools/paths.py:resolve_tool_binary` 3 级 fallback:

```python
# 1) PATH (shutil.which)
# 2) extend-tools/bin/<platform>/<name>.exe   (flat layout, e.g. file/exiftool/7zr/foremost/binwalk)
# 3) extend-tools/bin/<platform>/<name>/<name>.exe   (subdir fallback, steghide Cygwin)
```

**bug**: subdir fallback (line 89-95) 只看 `<name>/<name>.exe` 模式, 不支持 `<other>/<name>.exe`。

### 1.4 决策点 (Owner 2026-06-29 22:07 拍板 A)

> "A. 推 journal_clear → 我开 v0.5-extend-tools-subdir-flexible spec 修 xxd"

**修法** (2 步, 路径设计 A 选):

1. **`manifest.yaml` 加 3 块** (truth source, 手动维护):

   - 未知 name 走 PATH 优先 → fallback 仍 None

## 2. 改动

### 2.1 paths.py 改动 (~15 行)

```python
# 旧 (line 89-95):
sub_candidate = bindir / name / f"{name}{exe_suffix()}"
if sub_candidate.exists():
    return str(sub_candidate)
return None

# 新 (line 89-110):
sub_candidate = bindir / name / f"{name}{exe_suffix()}"  # 旧 steghide 兼容
if sub_candidate.exists():
    return str(sub_candidate)
# 异名 subdir fallback (per v0.5-extend-tools-subdir-flexible, Owner 22:07 拍板):
# 扫 bindir 下所有 subdir, 找 <subdir>/<name>{.exe}
# 典型用例: vim92/xxd.exe / vim92/diff.exe
# 不递归 (避免深度爆炸), 一层 subdir only
try:
    for sub in bindir.iterdir():
        if not sub.is_dir() or sub.name.startswith("."):
            continue
        sub_other = sub / f"{name}{exe_suffix()}"
        if sub_other.exists():
            return str(sub_other)
except OSError:
    # bindir 不存在 / 权限问题, fallback 到 None
    pass
return None
```

### 2.1.1 adapter 同步改动 (per v0.5-platform-extend-tools PR2 漏改, 5 个改了 12 个漏)

**实战发现** (跑 `pytest tests/unit/tools/shared/test_xxd.py` 仍 FAIL): `paths.py:resolve_tool_binary` 修了 xxd → `vim92/xxd.exe` 找到, **但** `xxd adapter` 调的是 `[self.binary_path or "xxd", ...]` 裸字符串 (line 50), **没**用 `resolve_tool_binary`. subprocess `xxd` 走 PATH lookup 找不到, 仍 exit 127.

**根因**: v0.5-platform-extend-tools PR2 改 5 个 adapter 用 `resolve_tool_binary` (binwalk/exiftool/foremost/sevenz/sevenz_extract), **漏改 12 个**走 `self.binary_path or "<name>"` 裸模式的:

| Adapter | 命令 | Win 装了? | 本次修? |
|---|---|---|---|
| `xxd` (shared/xxd.py:50) | xxd | ✅ vim92 | ✅ 必修 |
| `steghide_audio` (steghide_audio.py:44) | steghide | ✅ Cygwin build | ✅ 必修 |
| `strings` (shared/strings.py:56) | strings | ✅ Win 自带 (C:\Windows\System32\strings.EXE) | ⚠️ 可选 (Win 自带 OK) |
| `ffmpeg_video` / `ffmpeg_audio` / `ffprobe` | ffmpeg/ffprobe | ❌ | ⏳ 等 v0.5+ 评估装 |
| `sox` | sox | ❌ | ⏳ 等 v0.5+ 评估装 |
| `unzip` | unzip | ❌ (Win 用 7z 替代) | ⏳ 等 v0.5+ 评估装 |
| `tshark` / `tcpdump` / `protocol_router` (tshark) | tshark/tcpdump | ❌ | ⏳ 等 v0.5+ 评估装 |
| `grep` (Win 用 Select-String) | grep | ❌ | ⏳ 等 v0.5+ 评估装 |

**修法 2 个** (per 必装 2 个):
- `xxd.py:50`: `[self.binary_path or "xxd", ...]` → `[self.binary_path or resolve_tool_binary("xxd") or "xxd", ...]`
- `steghide_audio.py:44`: `[self.binary_path or "steghide", ...]` → `[self.binary_path or resolve_tool_binary("steghide") or "steghide", ...]`
- pattern 参考 `binwalk.py:90-97` (per v0.5-platform-extend-tools PR2)

**不动**其余 10 个 (Win 没装, 修了也调不到, 等 v0.5+ 实战触发再补).

变更: **+8 行 / -4 行** (2 个 adapter 各 1 行改 + 1 行 import + 1 行注释)

## 5. 风险

| 风险 | 概率 | 缓解 |
|---|---|---|
| 一层 subdir 扫到大量不相关目录 (e.g. exiftool_files/) 误匹配 | 低 | `exiftool_files/` 是普通目录不是 binary subdir, 无 `<name>.exe` 直接放; 加上 `if not sub.is_dir(): continue` 过滤 |
| 性能: iterdir 扫 5-10 subdir O(n) | 极低 | n < 20, OS 文件 IO 几乎免费, 比 shutil.which 还快 |
| 递归 subdir 风险 (e.g. vim92/runtime/xxd.exe 误匹配) | 中 | **不**递归, 1 层 only, 代码注释明确 |
| 跟现有 steghide 兼容路径冲突 | 低 | 旧路径 `<name>/<name>` 先查, 找到直接 return; 新通配是 fallback |
| manifest.yaml path_in_vendor 字段不存 | 中 | 字段是建议, paths.py 实际不读 manifest (manifest 是 truth source 但 paths.py 是 fast lookup), 不强制; 留 §2.2 注释说明 |
