## 2. 实施结果

### 2.1 pcapfix ⚠️ 路径偏差

**问题**：`brew install pcapfix` 失败 — Homebrew core 没有 `pcapfix` formula（只有 `pcapmirror`，那是流量镜像工具，不是修复器）。

**Workaround**（已落地）：
1. `curl https://sourceforge.net/projects/pcapfix/files/latest/download -o /tmp/pcapfix.zip`（已下载 98KB zip）
2. 解压得到 `pcapfix-1.1.6-win/`（含源码 + win .exe + Makefile）
3. `make` 编译成功（27 个 format warning 不影响功能）
4. `install -pDm755 /tmp/pcapfix-src/pcapfix /usr/local/bin/pcapfix`

**验证**：`pcapfix 2>&1 | head -2` 输出 `pcapfix 1.1.6 (c) 2012-2021 Robert Krause` ✅

**清理**：`/tmp/pcapfix.tar.gz` + `/tmp/pcapfix-1.1.6-win/` + `/tmp/pcapfix-src/`（源码拷）保留在 /tmp，下次装机复用

### 2.2 aircrack-ng ✅

`brew install aircrack-ng` 装好 1.7_2（bottled）。第一次 brew 命令超时但 install 实际完成；第二次 `brew install --dry-run` 确认 `Installed (on request)`。

**验证**：`aircrack-ng --help` 输出 `Aircrack-ng 1.7` ✅

### 2.3 scapy ✅

`pip3 install --break-system-packages scapy`（macOS 3.14 系统 Python，无 venv per `AGENTS.md §2.3`）

**依赖连带装**：pycryptodomex-3.23.0 + click-8.4.1（impacket 也需要，共享）

**验证**：`python3 -c "import scapy.all as sc; print(sc.conf.version)"` 输出 `2.7.0` ✅

### 2.4 impacket ✅

同一条 `pip3 install --break-system-packages impacket`

**依赖连带装**：flask-3.1.3 + ldap3-2.9.1 + ldapdomaindump-0.10.0 + pyOpenSSL-26.3.0 + cryptography-49.0.0 + dnspython-2.8.0 + werkzeug-3.1.3 + jinja2-3.1.6 + pyasn1-0.6.3 + pyasn1_modules-0.4.2 + itsdangerous-2.2.0 + blinker-1.9.0 + charset_normalizer-3.4.7 + markupsafe-3.0.3 + cffi-2.0.0 + pycparser-3.0 + six-1.17.0（共 16 个间接依赖）

**验证**：`python3 -c "from impacket.examples import smbclient"` 输出 `impacket OK` ✅

### 2.5 python-evtx ✅ 包装好，但 ⚠️ CLI shim 坏

**包状态**：`pip3 show python-evtx` 显示 **0.8.1** 已装在 `/Users/minzhizhou/Library/Python/3.14/lib/python/site-packages`（之前项目残留，命中）

**Python 模块 OK**：`import Evtx; import Evtx.Views` 通过 ✅

**⚠️ CLI `evtx_dump` 挂了** — `which evtx_dump` 指向 `~/.pyenv/shims/evtx_dump`，实际是 pyenv 3.13.5 的旧版本残留，shell 内容是：
```python
from scripts.evtx_dump import main
```
新版 0.8.1 的 `Evtx/` 包没有 `scripts/`，所以 `evtx_dump --help` 抛 `ModuleNotFoundError: No module named 'scripts'`。

**这不是今天要装的内容**，但因为 `tools.md §6.1 P0` 第 16 项是 `evtx_dump`，需要单独开 fix 迭代处理（见 §3）。
