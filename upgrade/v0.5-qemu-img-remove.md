```
                   22:58 拍 "装 qemu-img" → 新建 v0.5-qemu-img-extend-tools + v0.5-qemu-img-adapter

```

**这是合理的决策反转**, 两次都有实战依据：
- v0.5-train-018 (7z 26.02): 7z 不支持 → 装 qemu-img
- v0.5-train-019 (7z 23.01): 7z 支持 → 删 qemu-img

### 1.2 7z 版本差异 (per train-019)

| 7z 版本 | vmdk 支持 | 来源 |
|---|---|---|
| 17.05 | ✅ (per sevenz_extract spec 写错但实际可能对) | 历史 |
| 23.01 (Owner 当前) | ✅ **实测支持** | v0.5-train-019 本次 |
| 26.02 (Owner 之前装的) | ❌ 不支持 | v0.5-train-018 实战 |
| 24.x / 25.x | ❌ 大概率不支持 (回归) | 推测, 待 release notes 调研 |

**结论**: vmdk 支持是 7z 版本敏感功能, **7z 24.x ~ 26.x 某版本引入回归**, 23.01 及以下支持。

### 1.3 当前能力 vs 删除后

| 能力 | 现状 (qemu_img + qemu_img_extract + sevenz + sevenz_extract) | 删后 (sevenz + sevenz_extract + 7z 23.01) |
|---|---|---|
| **VMDK 格式探测** | qemu_img info → vdisk_format SP sev=3 | sevenz 检测 magic (VMDK magic 不是 7z magic, 是 KDMV/COWD, 可能漏报; 走 file 检测 VMware4) |
| **VMDK 真实解压** | qemu_img_extract → vdisk_extracted SP sev=5 (`qemu-img convert -O raw`) | **sevenz_extract** → archive_extracted SP sev=5 (`7z x vmdk_file.vmdk`) ✅ 实测 |
| **VHD / VDI / OVA / ISO** | qemu_img 统一处理 (但是 7z 23.01 多数不支持, 仅 qemu-img 能转) | **删除** (如果实战遇到再 v0.5+ 评估, 当前不在实战范围) |
| **zip / 7z / rar / tar / gz / bz2 / xz** | sevenz (不变) | sevenz (不变) |

**Net 变化**:
- ✅ Win 端 auto-run 少 1 个 500MB NSIS 安装包 (qemu-img 全 setup 包含 qemu-system GUI)
- ✅ 代码净简化 (删 2 adapter + 1 helper + 1 ps1 + 1 测试 + 6 处引用 + 文档同步)
- ⚠️ 失去 vdisk_format 专用 SP (VMDK 走 sevenz 时只是普通 archive_meta)
- ⚠️ 失去 VHD/VDI/OVA 支持 (如果实战遇到, 再 v0.5+ 评估 7z 23.01 或其他方案)

### 1.4 替代方案齐备

- ✅ sevenz adapter 已覆盖 zip/7z/rar/tar/gz/bz2/xz (v0.1+ 就有, 不变)
- ✅ sevenz_extract adapter 已覆盖真实解压 (v0.5-sevenz-extract, GUI 工具栏 "📦 7z 解压")
- ✅ 7z 23.01 完整安装实测支持 vmdk (v0.5-train-019 本次)
- ✅ file adapter 已识别 VMDK ("VMware4 disk image")

**结论**: 7z 23.01 完整覆盖 qemu_img_extract 的核心能力 (vmdk 真实解压), qemu_img info 可以被 file + strings + binwalk 替代 (VMDK descriptor 在 strings 里能看)。
