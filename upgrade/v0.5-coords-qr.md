## 2. 实施

### 2.1 新 decoder `core/decoders/coords_to_qr.py`

```
输入: text (坐标串) / file_path (--file 模式)
  ↓
_parse_coords(text) → [(r,c), ...]  (4 格式: 紧凑/带空格/无括号/换行/中文逗号)
  ↓
_infer_qr_size(coords) → int  (cell 索引 ≤65 / 像素级 >100)
  ↓
_infer_cell_px(coords, qr_size) → int  (像素级自动算 cell_px)
  ↓
_render_qr_png(coords, qr_size, out, cell_px) → PNG 写 input 同目录
  ↓
_run_zbar(png) → (stdout, decoded_lines)
  ↓
_find_flag(decoded) → flag_candidate
  ↓
DecodedQRResult (coords / qr_size / output_path / zbar_stdout / decoded / flag)
```

**关键设计**：
- **像素级 + cell 索引 双模式**：
  - cell 索引 (max ≤ 65): 每 cell 缩放 cell_px (默认 11)
  - 像素级 (max > 65): 直接用 1 像素 1 黑块，image = (max+1, max+1)
- **size 推断**：
  - ≤65：直接返回（已是 QR 模块数）
  - 100+：试除 21/25/29/33/... 找整除（raw / cell_px = 整数）
  - 兜底：min abs raw
- **samedir**：output_path 用 `output_path_for(file_path, suffix=".png", purpose="coords_qr")` (per v0.5-output-samedir)
- **text-based**：跟 hex-ascii 一样从 input 区走（不走 current_file），避免把图片当文本解

## 3. 真实样本端到端 (meihuai.jpg 真 flag 复用)

```bash
$ python -m automisc decode coords-qr --file /tmp/meihuai_coords.txt --keep
=== 🔳 坐标 → 二维码 ===
coords: [(7, 7), (7, 8), (7, 9), (7, 10), (7, 11), (7, 12), (7, 13), ... 34999 more]
qr_size: 65
output_path: /private/tmp/meihuai_coords__coords_qr.png
zbar_stdout: flag{40fc0a979f759c8892f4dc045e28b820}
zbar_decoded: ['flag{40fc0a979f759c8892f4dc045e28b820}']
flag_candidate: flag{40fc0a979f759c8892f4dc045e28b820}
width_px: 272
```

✅ **meihuai.jpg 真 flag `flag{40fc0a979f759c8892f4dc045e28b820}` 通过 coords-qr 全自动解出！**

`/tmp/meihuai_coords.txt` 是从 meihuai.jpg EOI 后 649566 字节 appended data 中解 hex 出的 ASCII 坐标串（per Owner 早期反馈："看起来左侧菜单栏中 hex 转 ascii 并不是真正的 hex->acsii"——其实是 **EOI 后 appended 的 16 进制 ASCII 串**，用 v0.5-hex-ascii-fix 解出 (7,7)\n(7,8)\n...，再喂给 coords-qr）。

完整 meihuai 链：**auto-run strings 找 appended hex → GUI Hex→ASCII 菜单解 (7,7)(7,8)... 形式 → GUI 坐标→二维码 菜单出 flag**。
