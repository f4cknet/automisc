  + def _maybe_suggest(tool_name, suspicious_points, file_path) -> list[SuspiciousPoint]:
      """对 auto_run 命中的 SP, 加 1 条 suggest SP (severity=4)."""
      suggests = []
      for sp in suspicious_points:
          # key 构造: tool_name + sp.category (简化版, 不细分 offset / description)
          key = f"{tool_name}:{sp.category}"
          # binwalk file_header 比较特殊, sub-category 靠 matched_pattern
          if tool_name == "binwalk" and sp.category == "file_header":
              # 从 matched_pattern 抽 sub-category (e.g. "ZIP archive" / "pyc")
              if "ZIP" in sp.matched_pattern:
                  key = "binwalk:file_header_zip"
              elif "7z" in sp.matched_pattern:
                  key = "binwalk:file_header_7z"
              elif "RAR" in sp.matched_pattern:
                  key = "binwalk:file_header_rar"
              elif "pyc" in sp.matched_pattern.lower():
                  key = "binwalk:file_header_pyc"
              else:
                  continue  # 其他 file_header 不加 suggest
          if key in _SUGGEST_MAP:
              cat, text = _SUGGEST_MAP[key]
              suggests.append(SuspiciousPoint(
                  id="",
                  tool_name="auto_run_suggest",
                  file_path=file_path,
                  category=cat,
                  offset=sp.offset,
                  matched_pattern=text,
                  severity=4,  # info, 不是真正可疑
                  suggested_action=(
                      "看 auto_run 命中的 SP 决定下一步, "
                      "GUI Run→Chain 弹 dialog 跑对应 chain"
                  ),
              ))
      return suggests

### 3.2 渲染流程 (新行为)

```
1. Owner 拖 PNG → main_window.dropEvent
2. find_suspicious_runner.start(file_path)
3. FindSuspiciousRunner.run:
   for tool_name in FIND_SUSPICIOUS_PICTURE_TOOLS:
       result = core.run_tool(tool_name, file_path)
       # 新增: 命中后自动加 suggest SP
       suggests = _maybe_suggest(tool_name, result.suspicious_points, file_path)
       result.suspicious_points.extend(suggests)
       results.append(result)
4. journal 累积:
   - zsteg SP: 3 条 lsb_text (severity=4)
   - zsteg suggest SP: 1 条 (severity=4, "建议手工跑 lsb-bytes chain")
   - strings SP: 3 条敏感关键词 (severity=5)
   - strings suggest SP: 1 条 (severity=4, "建议手工跑 bruteforce")
5. Owner 看 journal_panel:
   - severity=5: 真可疑 (e.g. 命中 password 字符串)
   - severity=4: lsb_text / suggest (info, 不必立刻看)
   - severity=2: usage_hint (e.g. PIL 缺失)
6. Owner 决策:
   - 看到 "zsteg suggest: 建议跑 lsb-bytes chain" → 点 Run→Chain→lsb-bytes → 弹 dialog → 跑
   - 看到 "binwalk suggest: ZIP" → 点 Run→Chain→zip → 跑
   - 看到 "strings suggest: 密码" → 用密码手工试
```

### 3.3 关键设计决策

#### 3.3.1 suggest SP severity=4 (info, 不是真正可疑)

per v0.5-auto-run-discipline 铁律 "可疑点越多越好":
- 真可疑 (e.g. binwalk 命中 pyc) severity=5
- suggest 提示 (e.g. "建议跑 lsb-bytes chain") severity=4
- 命中字符串 severity=5 (per 关键词白名单)
- usage_hint (e.g. PIL 缺失) severity=2

severity=4 让 suggest 显眼但不"真可疑"刺眼 — Owner 知道这是提示。

#### 3.3.2 dedup: 每个 tool + 每个 category 只加 1 条 suggest

如果 zsteg 命中 3 条 lsb_text SP, 不加 3 条 suggest (噪声); 加 1 条 suggest (足够引起 Owner 注意)。

#### 3.3.3 不动 auto_run 池 (per 铁律 7)

撤回 v0.5-lsb-bytes-auto-run 后, auto_run 池保持 6 tools (zsteg / stegseek / exiftool / binwalk / strings / file)。
本 spec 只加"命中后 suggest"逻辑, **不动** auto_run 池本身。

## 7. 待 Owner 拍板的决策点

### Q1: suggest SP severity 用多少?

| 选项 | severity | 优点 | 缺点 |
|---|---|---|---|
| **A** | 4 (info) | 显眼但不刺眼 | 跟"真可疑" severity=5 区分 |
| B | 5 (跟命中一样) | 显眼 | 跟真可疑混淆 |
| C | 3 (low info) | 安静 | 不显眼, Owner 可能漏 |

**Mavis 建议 A** — severity=4 显眼但不刺眼, 让 Owner 注意到。

### Q2: suggest SP category 用什么?

| 选项 | category | 优点 | 缺点 |
|---|---|---|---|
| **A** | "auto_run_suggest" | 单一 category 便于过滤 | 不区分命中类型 |
| B | "auto_run_suggest_<tool>" (e.g. "auto_run_suggest_zsteg") | 细分 | category 字符串膨胀 |

**Mavis 建议 A** — 单一 category, 后续 v0.5+ 可以细分 (per §4.2 OUT)。

### Q3: 覆盖范围?

| 选项 | tool:category | 数量 | 优点 | 缺点 |
|---|---|---|---|---|
| **A** | zsteg:lsb_text + binwalk:file_header_{zip,7z,rar,pyc} + strings:敏感关键词 | 6 条 | MVP, 覆盖 Owner 实战高发场景 | 不全 |
| B | + stegseek / exiftool | 10+ 条 | 覆盖广 | 过度设计 |
| C | A + 自动从 SP category 推导 suggest (智能) | 全自动 | 智能化 | 复杂, 误报 |

**Mavis 建议 A** — MVP 6 条覆盖高发, 后续实战再加。

## 8. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| suggest SP 太多刷屏 | dedup (per 工具 per category 只 1 条 suggest) |
| suggest 文本误导 Owner (e.g. binwalk 命中 PNG suggest 跑 zip) | _SUGGEST_MAP 写死, 不动态推导 |
| suggest 跟现有 SP category 重复 (e.g. owner 已经有 zip_chain 工具栏) | suggest severity=4 + category=auto_run_suggest 区分 |
| auto_run 跑超 5s (suggest 加逻辑后) | suggest 是 dict lookup + list extend, O(N), < 1ms |
| 未来 tool 加新 category, suggest 不覆盖 | MVP 6 条够用, 实战 Owner 反馈再加 |
