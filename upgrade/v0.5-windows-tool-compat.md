## 3. 影响面

### 3.1 exiftool adapter（Bug 2 修复）

**文件**：`src/automisc/tools/shared/exiftool.py`

**改动**：在 `cmd` 构建时追加 `-charset utf8` 参数。

```python
# 当前 (line ~XX):
cmd = [self.binary, "-j", "-G1", "-a", "-s", str(file_path)]

# 修复后:
cmd = [self.binary, "-charset", "utf8", "-j", "-G1", "-a", "-s", str(file_path)]
```

**改动量**：~1 行 + 单测更新（断言 EXIF 中文字段输出预期）

### 3.2 file adapter（Bug 3 修复）

**2 选 1（Q2）**：

**方案 A · 装 magic-win prebuilt `file.exe`**（推荐）：
- 下载 [magic-win file.exe](https://github.com/nscaife/file-windows/releases) ~3MB
- 放到 `extend-tools/bin/win-x64/file.exe`
- 更新 `extend-tools/manifest.yaml` 加 `file` 工具条目
- `tools/paths.py:resolve_tool_binary("file")` 已走 PATH → extend-tools fallback，不用改

**方案 B · `file` adapter 走 `python-magic-bin`**：
- `pip install python-magic-bin`
- 改 `src/automisc/tools/shared/file.py` 走 Python 包
- 改动量 ~10 行（subprocess → magic.from_file）

**改动量**：方案 A ~5 行（manifest + 测试），方案 B ~15 行（adapter 重构）

### 3.3 stegseek / zsteg unavailable 标记（Bug 1 修复 · steghide 已装不入白名单）

**改动**：

1. **GUI menu marker**：`gui/menu_dock.py` 加 `_BIN_UNAVAILABLE_ON_WIN = {"stegseek", "zsteg"}`（zsteg 已在 v0.5-lsb-detector 移除 auto-run，加白名单兜底），渲染菜单时检查 platform == "win32" + 名字在白名单 → marker `✗` + tooltip "Windows unavailable"
2. **auto-run 白名单**：`gui/auto_runner.py:FIND_SUSPICIOUS_PICTURE_TOOLS`（或其他 auto-run 注册处）加白名单检查，`if sys.platform == "win32" and tool.name in UNAVAILABLE_ON_WIN: skip`
3. **stegseek adapter description**：`src/automisc/tools/steganography/audio/stegseek.py` 加注释 "Windows unavailable (Linux-only C++ tool)"

> **steghide 不入白名单** — `extend-tools/bin/win-x64/steghide/steghide.exe` 已在 PR3-prep (`b8e5241`) 实装（v2.6 tools.md 标 ✅），Windows 上正常用。

**改动量**：~20 行（GUI marker + auto-run 跳过 + adapter 注释）

### 3.4 测试

**新增单测**：
- `tests/unit/tools/shared/test_exiftool.py`：加 1 个 case 断言 EXIF 中文输出（构造 EXIF 含 UTF-8 字节 → 调 adapter → assert 字符串不是 `å›¾` 乱码）
- `tests/unit/tools/shared/test_file.py`：加 1 个 case 断言 Win 上 `file` binary 找到（方案 A）/ magic.from_file 调通（方案 B）
- `tests/unit/gui/test_unavailable_marker.py`：新文件，断言 GUI marker 在 win32 + stegseek / zsteg 组合下渲染 `✗`（steghide 已装不在白名单，不测）

**改动量**：~80 行单测

## 7. 风险 & 注意事项

- **PR2 装 binary 涉及网络下载** —— 跟 automisc"完全离线"产品定位**不冲突**（per v0.5-platform-extend-tools §3：install 脚本首次下 binary 是开发期工具，用户跑 GUI 仍离线）。但 owner 需保证 `extend-tools/install.ps1` 幂等（已下载跳过）
- **magic-win binary 来源可信度** —— GitHub `nscaife/file-windows` 是 [tridactyl/file-windows fork](https://github.com/tridactyl/file-windows) 社区维护，需 owner 验证 source 后再加 manifest URL + sha256
- **`-charset utf8` 影响所有 EXIF 字段输出** —— 包括 macOS 跑 exiftool 也会强制 UTF-8（跟 macOS 默认一致，无回归风险）
- **auto-run 白名单跨平台行为** —— `if sys.platform == "win32"` 检查必须明确，避免 macOS 上误跳过 stegseek（macOS 上 stegseek 是装的）；steghide 不在白名单,Win/macOS 都正常跑
