## 1. 背景

### 1.1 问题陈述 (per Owner 06-21 11:54 反馈)

v0.5-pyc-magic-sniffer 实施后, pyc_decompiler decoder **注册成功 + CLI 跑通 + 反编译出 N=NP 题源码**, 但 **GUI 左侧工具栏看不到 "🐍 Pyc 反编译" 入口**。

### 1.2 根因

`pyc_decompiler.py` 注册到 `core.decoders.registry` 时, `group="general"` (per v0.5-decoder-menu 默认)。但 `gui/menu_dock.py:_get_cipher_categories_from_registry()` 只渲染 `group != "general"` 的 decoder (per v0.5-cipher-decoders 设计, 用来区分"普通 decoder"和"cipher 解密工具")。

```
pyc_decompiler group=general → 不渲染到 "🔤 解密工具1" → GUI 工具栏看不到
```

### 1.3 Owner 决策

> "**大多数 pyc 反编译时 python2,把这个放到 gui 左边工具栏中**"

含义:
1. **GUI 工具栏必须有"🐍 Pyc 反编译"入口** (per v0.5-cipher-decoders 治理: 操作类工具必须显示菜单)
2. **默认走 Python 2.x 反编译路径** (uncompyle6) —— 这一点 `run_pyc_decompiler` 已经实现 (`is_py2 → uncompyle6`)

---

## 2. 目标

### 2.1 能力 F: pyc_decompiler 显示在 GUI 左侧工具栏 (per Owner 06-21 11:54)

**目标**: 把 `decoder:pyc_decompiler` 加到 `gui/menu_dock.py:TOOL_CATEGORIES` 固定分类,GUI 启动时自动渲染到工具树。

### 2.2 选 display 分类

3 个选项:

| 选项 | 分类标题 | 优点 | 缺点 |
|---|---|---|---|
| **A** | `🐍 反编译工具 (v0.5-pyc-magic-sniffer)` | 新建独立分类,清晰 | 多一个分类,GUI 树更深 |
| **B** | `🔓 解码工具 (v0.5+ Decoders)` (现有) | 复用现有分类,不新增 | pyc_decompiler 跟 base64-image 一起,语义混 |
| **C** | `🔤 解密工具1 (v0.5-cipher-decoders)` | 复用 cipher 分类(自动渲染) | pyc_decompiler 不是 cipher,语义错位 |

**Owner 没明示分类, Mavis 建议 A** —— 新建独立分类,语义清晰, 跟 v0.5-pyc-magic-sniffer 绑定。

### 2.3 默认走 Python 2.x (已实现)

`run_pyc_decompiler` 当前逻辑:
```python
if is_py2:
    source, method = _decompile_with_uncompyle6(file_path)  # Py2.x 走 uncompyle6
elif is_py3:
    source, method = _decompile_with_decompyle3(file_path)  # Py3.x 走 decompyle3
```

**per Owner 决策已经实现, 不动**。但 spec 注明: **dis fallback** 在前两者都失败时跑 (兜底)。

```

```

**只改 `menu_dock.py`**, 不动其他 GUI 文件:
- `main_window.py`: 不用改, GUI 启动自动走 `ToolMenuDock._populate` 渲染
- `decode_runner.py`: 不用改, `decoder:pyc_decompiler` 已经走 `_run_decoder` 路径
- `pyc_decompiler.py`: 不用改, group="general" 保持 (这是 cipher 分类的过滤条件, 不影响 TOOL_CATEGORIES 渲染)

### 3.2 渲染流程验证

`gui/menu_dock.py:_populate()`:
1. 取 `TOOL_CATEGORIES` 固定分类 + cipher 自动分类
2. 对每个 tool, 检查 `ADAPTER_TOOLS` 决定 marker (✓/✗)
3. pyc_decompiler 不在 `ADAPTER_TOOLS` (它是 decoder, 不是 adapter), 所以 marker = "✓" (默认 decoder 走通)

✅ 自动显示。
