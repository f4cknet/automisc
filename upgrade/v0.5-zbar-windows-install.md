## 1. 背景

`src/automisc/tools/misc/brainteaser/zbar.py` 写好多年, GUI 工具栏 "🔳 二维码解析" (per `v0.5-more-converts` 重命名) 也早渲染好, 但 Win 端 binary 从未装:
- `tools.md §6.1 #19` 标 ❌ pending
- `extend-tools/manifest.yaml` win-x64 没列 zbar
- `extend-tools/install.ps1` 没加 zbar
- 实际效果: Owner 点 "🔳 二维码解析" → adapter 调 `zbarimg` → 找不到 binary → exit 127 → 报 "executable not found"

2026-06-28 22:58 Owner 拍板 "安装zbar" → AI 试 `pip install zbar` → **失败**:

```
× Building wheel for zbar (pyproject.toml) did not run successfully.
  error: Microsoft Visual C++ 14.0 or greater is required.
```

**根因**: PyPI 上 `zbar` (0.10) 是 2009 年的 Py2 时代 C 扩展包, Windows 历来无 wheel, 源码要 MSVC 编译, 现代 Python 装不上。

## 2. 调研: Windows zbar 部署方案

| 方案 | 可行性 | 备注 |
|---|---|---|
| **A. PyPI `zbar` 0.10 (老)** | ❌ | Py2 era C 扩展, Win 无 wheel, 要 MSVC 编译 |
| **B. SourceForge `zbar-0.10-setup.exe`** | ⚠️ | 2010 NSIS installer, 官方有但**直接下载链接全被 SF 拦截**返 HTML (实测 2026-06-28 4 个 mirror 全 fail) |
| **C. MSYS2 `mingw-w64-x86_64-zbar`** | ⚠️ | 需装整个 MSYS2 (~500MB), 不在项目范围 |
| **D. `pyzbar` (Py3 ctypes wrapper, Win wheel 自带 zbar DLL)** | ✅ | **本 spec 选这个** |
| **E. 自实现 zbarimg 兼容层** | ❌ | zbar 30+ 格式识别 + 复杂图像处理, 半年工作量 |

**选 D 的关键事实** (per PyPI pyzbar 0.1.9 + 实测 2026-06-28):
1. `pyzbar` PyPI 描述: *"The zbar DLLs are included with the Windows Python wheels."*
2. 安装后 `<python>/Lib/site-packages/pyzbar/` 自带 2 个 DLL:
   - `libzbar-64.dll` (164 KB, libzbar 主库)
   - `libiconv.dll` (958 KB, GNU libiconv 字符集转换)
3. `pyzbar.pyzbar.decode(PIL.Image)` API 等价于 `zbarimg --raw` 行为 (一行一条解码文本)

## 3. 二次失败 + 根因 (per Owner 2026-06-28 23:05 反馈)

`pip install pyzbar` 本身 OK (817KB wheel, 6.0 MB/s 下完), 但**实际 import 失败**:

```
File "pyzbar/zbar_library.py", line 51, in load_objects
  cdll.LoadLibrary(str(directory.joinpath(dep)))
FileNotFoundError: Could not find module 'libiconv.dll' (or one of its dependencies).
```

**根因分析** (per `pefile` dump `libzbar-64.dll` PE imports):
- `libzbar-64.dll` imports `MSVCR120.dll` (VS 2013 C runtime) 大量函数: `memcmp/memcpy/strdup/__iob_func/__clean_type_info_names_internal/_calloc_crt/_initterm_e/...`
- 系统只装了 VS 2015+ runtime (vcruntime140 / msvcp140), 缺 **VS 2013 runtime** (msvcr120 / msvcp120)
- libiconv 在 pyzbar site-packages 自己有, 加载 OK
- 但 libzbar-64 链了 MSVCR120, 找不到 → 整个 pyzbar 加载 fail

**修复**: 把 `msvcr120.dll` (973 KB, 微软允许随 redist 分发) 复制到 pyzbar site-packages, libzbar 即可加载。  
**实测**: copy 后 pyzbar import OK, `decode()` 跑通 `sample_qr_flag.png` → `flag{pr8_smoke_qr_xyz}` ✓。

## 5. 实施细节

### 5.1 zbar adapter 重构 (per 4.1 (1))

```python
"""zbar adapter (per `tools.md` §3.11 + v0.5-zbar-windows-install)

`zbarimg`: zbar 项目的图片扫描 CLI, 支持 QR / EAN-13 / Code-128 / PDF417 / DataMatrix 等 30+ 条码格式.

**v0.5-zbar-windows-install**: 从 subprocess `zbarimg` 切到 pyzbar Python lib.
- zbar 0.10 官方 Windows binary (SourceForge NSIS installer) 下载链接失效 (4 mirror 全 200 HTML)
- pyzbar 0.1.9 Win wheel 自带 libzbar-64.dll + libiconv.dll
- output 格式跟 `zbarimg --raw` 100% 兼容 (一行一条解码文本), GUI 入口/单测**不动**
- macOS: pyzbar 仍能 work, 但 brew zbarimg 也 OK (优先 PATH 命中)

**Win ship msvcr120.dll**: libzbar-64.dll 是 VS 2013 编译, 链 MSVCR120.dll, 系统若没装 VS 2013 redist
会报 "Could not find module 'libiconv.dll' (or one of its dependencies)" (实测 2026-06-28).
install.ps1 把 extend-tools/bin/win-x64/msvcr120.dll 复制到 pyzbar site-packages, 修这个 issue.
"""
from __future__ import annotations

import re
from typing import List

from automisc.core.registry import register_tool
from automisc.core.result import ToolResult
from automisc.core.suspicious import SuspiciousPoint, scan_output_for_suspicious
from automisc.tools.base import ToolAdapter


# zbarimg --raw 输出格式: 一行一条解码文本 (无前缀)
# 我们的 pyzbar 实现保持同样格式
_ZBAR_OUTPUT_RE = re.compile(r"^(?:([\w-]+):)?(.+)$")


@register_tool
class ZbarAdapter(ToolAdapter):
    """`zbarimg` adapter — QR / 条码识别 (per v0.5-zbar-windows-install 走 pyzbar 后端)."""

    name = "zbar"
    category = "misc_brainteaser"
    description = "QR / 条码识别 (pyzbar 后端, 30+ 格式: QR / EAN-13 / Code-128 / PDF417 / DataMatrix)"

    default_timeout = 30.0

    def check_available(self) -> bool:
        """v0.5-zbar-windows-install: check pyzbar importable (not binary in PATH)."""
        try:
            import pyzbar.pyzbar  # noqa: F401
            return True
        except (ImportError, OSError):
            return False

    def run(self, file_path: str) -> ToolResult:
        # v0.5-zbar-windows-install: use pyzbar instead of subprocess zbarimg
        # 行为等价: 30+ 格式解码, output 格式 = zbarimg --raw (一行一条文本)
        try:
            from PIL import Image
            from pyzbar.pyzbar import decode as pyzbar_decode
        except ImportError as e:
            return ToolResult(
                tool_name=self.name,
                exit_code=127,
                stdout="",
                stderr=f"pyzbar not installed: {e} (run: pip install pyzbar)",
                suspicious_points=[],
            )

        try:
            img = Image.open(file_path)
            results = pyzbar_decode(img)
        except FileNotFoundError:
            return ToolResult(
                tool_name=self.name,
                exit_code=2,
                stdout="",
                stderr=f"file not found: {file_path}",
                suspicious_points=[],
            )
        except Exception as e:
            # PIL.UnidentifiedImageError for non-image files
            return ToolResult(
                tool_name=self.name,
                exit_code=1,
                stdout="",
                stderr=f"decode failed: {type(e).__name__}: {e}",
                suspicious_points=[],
            )

        # 构造 stdout: 跟 zbarimg --raw 行为一致 (一行一条解码文本)
        stdout_lines: List[str] = []
        for r in results:
            data_bytes = r.data
            try:
                data_str = data_bytes.decode("utf-8")
            except UnicodeDecodeError:
                # 二进制 payload (e.g. raw bytes) fallback
                data_str = data_bytes.decode("latin-1", errors="replace")
            stdout_lines.append(data_str)
        stdout = "\n".join(stdout_lines)

        # 复用原 zbar SuspiciousPoint 逻辑 (per tools.md §3.11)
        suspicious: list[SuspiciousPoint] = []
        # 1. 通用扫描 (捕获 flag{...})
        suspicious.extend(scan_output_for_suspicious(
            tool_name=self.name, file_path=file_path, stdout=stdout,
        ))

        # 2. 解析每条识别结果 — 跟原 zbar 行为 1:1 兼容
        for line in stdout.splitlines():
            line = line.strip()
            if not line:
                continue
            url_match = re.match(r"^(https?|ftp|file)://(.+)$", line)
            if url_match:
                code_type, content = url_match.group(1), url_match.group(2)
            else:
                m = _ZBAR_OUTPUT_RE.match(line)
                if not m:
                    continue
                code_type = m.group(1) or "unknown"
                content = m.group(2).strip()

            if not content:
                continue

            if len(content) > 50:
                suspicious.append(SuspiciousPoint(
                    id="", tool_name=self.name, file_path=file_path,
                    category="barcode_long_content", offset=None,
                    matched_pattern=f"{code_type}: {content[:120]!r} (len={len(content)})",
                    severity=2,
                    suggested_action="长字符串可能是 base64/URL/编码内容, 建议 base64/hex 解码或访问 URL",
                ))
            elif content.startswith(("/", "//", "?", "&")) or code_type in ("http", "https", "ftp", "file"):
                full_url = f"{code_type}:{content}" if not content.startswith("//") else f"{code_type}:/{content}"
                suspicious.append(SuspiciousPoint(
                    id="", tool_name=self.name, file_path=file_path,
                    category="barcode_url", offset=None,
                    matched_pattern=f"{code_type}: {full_url[:120]!r}",
                    severity=2,
                    suggested_action="URL 线索: 在浏览器访问或 curl 抓内容",
                ))
            else:
                suspicious.append(SuspiciousPoint(
                    id="", tool_name=self.name, file_path=file_path,
                    category="barcode_text", offset=None,
                    matched_pattern=f"{code_type}: {content[:80]!r}",
                    severity=1,
                    suggested_action="记录识别内容",
                ))

        # 3. 报告识别数量
        n = sum(1 for line in stdout.splitlines() if line.strip())
        if n > 0:
            suspicious.append(SuspiciousPoint(
                id="", tool_name=self.name, file_path=file_path,
                category="barcode_meta", offset=None,
                matched_pattern=f"识别 {n} 个条码/二维码",
                severity=1,
                suggested_action="记录识别数量",
            ))

        return ToolResult(
            tool_name=self.name,
            exit_code=0,
            stdout=stdout,
            stderr="",
            suspicious_points=suspicious,
            duration_ms=0,  # pyzbar 无独立计时, GUI 显示 0ms 可接受
        )
```
