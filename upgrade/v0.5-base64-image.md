## 1. Owner 规格

```
输入: 文件路径
  1. 读全文
  2. 预处理: 剥 data URL 头 (data:image/(jpeg|jpg|png|gif|webp|bmp);base64,) -> 记 mime
  3. base64 decode -> raw bytes
     失败 -> 报错
  4. 写 raw 到 tmp
  5. 用 `file` 命令检测 mime
     - image/* -> 成功
     - 非 image -> 报错"转图片失败"
```

## 2. 实施

### 2.1 新增 `src/automisc/core/decoders/base64_image.py`

| 决策点 | 实现 |
|---|---|
| data URL 头检测 | regex `_DATA_URL_RE` (RFC 2397) — `data:<mime>;base64,<data>` |
| 严格 base64 decode | `b64decode(s, validate=True)` 避免 validate=False 的 padding 误判 |
| Padding 容错 | `_strip_padding` 补齐 4 倍数 (e.g. `abc` → `abc=`) |
| 无 data 头 fallback | 试加 6 个常见 `data:image/*;base64,` 头 + 严格 decode; 都不行再试纯 base64 |
| 文件后缀推断 | mime → `.jpg`/`.png`/`.gif`/`.webp`/`.bmp`; fallback `.png` (给 `file` 提示) |
| `file` 命令验证 | `shutil.which("file")` + `subprocess.run(["file", "--brief", path])` |
| 输出 | `Base64ImageResult` (output_path / detected_mime / source_mime_hint / raw_size / kept_output) |
| 失败 | `Base64ImageError(Exception)` 含中文消息 |

### 2.2 新增 `automisc decode base64-image` CLI 子命令

```
$ automisc decode base64-image --file KEY.exe --keep --out-dir /tmp/b64
```
