## 2. automisc 行为对比

### Owner 报告：工具栏"zip 修复"按钮 + auto_run 都没解密

| 入口 | 行为 | 结果 |
|---|---|---|
| GUI 工具栏 "fix_pseudo_zip" 按钮 | 调 `FixPseudoEncryptionAction`（单 Action）| ❌ 失败（hexedit 假设错误）|
| `automisc run --tool unzip` | 调 `unzip adapter`（只 `unzip -l` 列表）| ❌ 0 SP（archive_meta 仅 1 file）|
| `automisc run --tool sevenz` | 调 `7z l` 列表 | ❌ "archive contains 2 files" 但不解压 |
| **`automisc chain --chain zip`** | **完整 DAG: try_unzip → fix_pseudo → bruteforce** | ✅ 1 步 `try_unzip` 成功，无密码直接解 |

### 关键差异：单 Action vs 完整 chain

- `FixPseudoEncryptionAction` 是**单 Action**：只调 hexedit 修改 flag bit 0 → 0x00
- `try_unzip` 在 chain 里用 `zipfile.ZipFile.extractall` 直接试——**Python stdlib 不强制密码**（密码空时直接解，密码非空时报错）
- **GUI 工具栏按钮绕过了 try_unzip 第一关**——所以"伪加密"识别不到

## 3. AI 兜底（per §5.3 step 3）

```bash
# Step 1: 列表 (跟 automisc unzip adapter 同行为)
$ unzip -l pseudo.zip
  Length      Date    Time    Name
        23  05-05-2017 20:26   flag.txt

# Step 2: 试空密码 (automisc 工具栏按钮没做这步)
$ unzip -P '' pseudo.zip -d /tmp/zip_test
  inflating: /tmp/zip_test/flag.txt

# Step 3: 拿 flag
$ cat /tmp/zip_test/flag.txt
flag{Adm1N-B2G-kU-SZIP}
```

**关键 insight**：伪加密本质是"加密位为 1 但无真加密 header"，`zipfile.extractall` 直接通过——CLI `unzip` 在没密码时报"需要密码"是因为它检测 encrypt flag 后强制要密码。

## 4. automisc 应补能力（架构升级输入）

| 能力 | 现状 | 应补 |
|---|---|---|
| **`FixPseudoEncryptionAction` GUI 入口** | 调单 Action (hexedit flag bit 0) | **改为调 `build_zip_chain_dag()`**（DAG 跑完整 try_unzip → fix_pseudo → bruteforce）|
| **auto_run top-5 推荐** | 推荐 `unzip` / `sevenz` / `john` 但不跑链 | 修：`.zip` 文件 → top-1 推荐 `chain:zip`（不 `unzip`）|
| **unzip / sevenz adapter** | 只列表不解压 | 改：默认 `unzip -o <file> -d <outdir>` 真正解压（per v0.5-output-samedir）|
| **解压后 SP 字段** | 没说"解压成功" | 解压成功 → SP 包含 `extracted_to` 路径 + journal 高亮 |
| **伪加密识别** | 靠链 try_unzip 试 | 保留（已经能识别），但工具栏按钮要走到这一步 |

## 5. 架构判定（per §5.2）

| 判定项 | 结论 |
|---|---|
| 同类样本 | **≥ 3 类**：（a）伪加密（这题）；（b）真加密但密码空；（c）真加密短密码爆破；（d）损坏 zip 修复 |
| 架构层 vs 单题 | **架构层**——"GUI 工具栏按钮 vs CLI 完整 chain 行为不一致"是 UX 架构问题，**不是单题特例**（所有走工具栏按钮 + 走 chain 的入口都有这个差异）|
| 触发架构升级 | ✅ **是** → 触发 `upgrade/v0.5-zip-toolbar-routing.md`（待建）|
| 训练日志直接改代码 | ❌ **否**（per §5.5）——只描述问题，架构升级走单独迭代 |
