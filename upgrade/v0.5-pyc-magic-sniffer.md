## 1. 背景

### 1.1 问题陈述 (per 06-21 11:35 Owner 实测反馈)

跑 `automisc chain --chain lsb-bytes --channel G --bit 0 --scan-order col --file image.png` 后:
- ✅ 抽出 130625 字节,字节流 **magic `03 f3 0d 0a` = Python 2.7 magic (62211)** = **合法 pyc**
- ⚠️ 跑 `automisc decode magic_sniffer` → **0 hits**(字典缺 Py2.7 magic 模式)
- ❌ automisc 没有"反编译 .pyc"功能(Owner 手工用 `uncompyle6` 反编译得到 KEY1/KEY2 → 解 flag)

### 1.2 automisc 现有能力 gap

| 维度 | 现状 | writeup N=NP 题需要 | 缺口 |
|---|---|---|---|
| magic 字典覆盖 | 39 entry(PNG/ZIP/pyc/JPEG/ELF/WASM/Mach-O 等) | Py2.x magic 模式 `03 f3 0d 0a` 等 | ❌ 缺 Py2.x 系列 |
| pyc 反编译 | 无,需手工调 `uncompyle6` / `decompyle3` | 解出 KEY1/KEY2 → 反推 flag | ❌ 缺 pyc_decompiler decoder |
| flag 解密 | 无(per Owner 06-21 11:40 Q3 = 先不做) | 不同题加密算法不同,不强求通用 | ✅ 暂不做 |

## 2. 目标 (Owner 06-21 11:40 拍板)

### 2.1 能力 D: 扩展 magic_sniffer magic 字典(per Q1=Y)

**目标**: 加 **8+ Py2.x magic + 5+ Py3.x magic 变体** 到 `EXTENDED_MAGIC_SIGNATURES`,让 magic_sniffer 能自动识别 Py2.7 / Py3.x 系列 pyc 文件。

### 2.2 能力 E: `pyc_decompiler` decoder(per Q2=Y)

**目标**: 新建 `core/decoders/pyc_decompiler.py`,封装 `uncompyle6` / `decompyle3` / `xdis` 反编译逻辑:
- 输入: .pyc 文件路径
- 输出: Python 源码字符串 + 元数据(原始 magic int / Python 版本 / 时间戳)
- 注册到 decoder registry,CLI/GUI 自动从 registry 渲染

### 2.3 不做(per Q3=先不做)

❌ `pyc_decrypt` decoder —— 不同题目加密算法不同,不强求通用。

## 3. 设计

### 3.1 模块扩展 / 新增

```
src/automisc/core/decoders/magic_sniffer.py
  - EXTENDED_MAGIC_SIGNATURES 加 13+ entry:
    Py2.4 (62061=0xf23b) → b'\x3b\xf2\x0d\x0a'
    Py2.5 (62071=0xf261) → b'\x61\xf2\x0d\x0a'
    Py2.6 (62081=0xf281) → b'\x81\xf2\x0d\x0a'
    Py2.7 (62211=0xf303) → b'\x03\xf3\x0d\x0a'  ← image.png 这题用
    Py3.0 (3000=0x0bb8) → b'\xb8\x0b\x0d\x0a'
    Py3.1 (3150=0x0c4e) → b'\x4e\x0c\x0d\x0a'
    Py3.2 (3360=0x0d20) → b'\x20\x0d\x0d\x0a'
    Py3.3 (3375=0x0d2f) → b'\x2f\x0d\x0d\x0a'
    Py3.4 (3400=0x0d48) → b'\x48\x0d\x0d\x0a'
    Py3.5 (3410=0x0d52) → b'\x52\x0d\x0d\x0a'
    Py3.6 (3420=0x0d5c) → b'\x5c\x0d\x0d\x0a'
    Py3.7 (3394=0x0d42) → b'\x42\x0d\x0d\x0a'
    Py3.8 (3400) → 同 3.4
    Py3.9 (3425=0x0d61) → b'\x61\x0d\x0d\x0a'
    Py3.10 (3439=0x0d6f) → b'\x6f\x0d\x0d\x0a'
    Py3.11 (3495=0x0da7) → b'\xa7\x0d\x0d\x0a'
    Py3.12 (3531=0x0dcb) → b'\xcb\x0d\x0d\x0a'
    (注: magic 字典已有 Py3.0/3.3/3.8/3.10/Py2.x 几个,补全剩余版本)

src/automisc/core/decoders/pyc_decompiler.py    ← 新 (能力 E 核心)
  - run_pyc_decompiler(file_path) -> PycDecompileResult
    1. xdis.load_module(file) → 拿 version + magic_int + co
    2. if Py2.x → uncompyle6.decompile_file
       elif Py3.x → decompyle3.decompile_file
       else → dis 反汇编 bytecode (fallback)
    3. 返回 source_code 字符串 + 元数据
  - register_decoder(DecoderSpec(
      name="pyc_decompiler",
      display="🐍 Pyc 反编译",
      category="decode",
      cli_cmd="decode pyc_decompiler",
      run=run_pyc_decompiler,
      description="Py2.x / Py3.x pyc 反编译到 Python 源码"
    ))

src/automisc/__main__.py
  - decoder 自动从 registry 注册(per v0.5-decoder-menu),无需改 CLI
```

## 8. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| `uncompyle6` 对 Py3.x 反编译支持差 | 主用 `decompyle3`(Py3.x 支持好),Py2.x 用 `uncompyle6`,fallback 用 `dis` 反汇编 |
| `magic_sniffer` magic 字典加太多 entry,影响性能 | 36 → ~50 entry,仍 O(N×max_offset),影响可忽略 |
| image.png 反编译出的源码可能 uncompyle6 部分有 bug | smoke 阶段用真实 image.png 验证,反编译失败就 fallback 到 `dis` 字节码 |
| 用户的 Python 环境没装 `uncompyle6` / `decompyle3` | v0.5-lsb-byte-stream-extract smoke 阶段已装;setup 文档说明 |
