## 3. 关键技术点

### 3.1 Base64 自定义表解码

**CTF 场景**：
- 标准 base64 表：`ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/`
- CTF 经常**打乱表**（替换字母顺序），需要用户**提供自定义表**

**算法**：
```python
def decode_base64_custom(s: str, custom_table: str) -> bytes:
    """custom_table 长度必须为 64（base64 标准表）或 65（含 '=' 填充）"""
    if len(custom_table) not in (64, 65):
        raise ValueError(f"custom_table length must be 64/65, got {len(custom_table)}")
    pad = '=' if len(custom_table) == 65 else None
    # 反向表: char -> 6-bit index
    trans = str.maketrans(custom_table, "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/")
    normalized = s.translate(trans)
    if pad:
        normalized = normalized.replace(pad, '=')
    return base64.b64decode(normalized)
```

**GUI 交互**（重要）：
- decoder 注册时标记 `interactive=True`
- 触发 decoder 时弹 `QInputDialog.getText()` 让用户输入自定义表（64 字符）
- 用户可粘贴完整的 64 字符串 OR 选 "常用变体"（base64 url-safe `A-Za-z0-9-_`、base62 字母表等）

### 3.2 ROT 系列对照

| 名称 | 范围 | 公式 |
|---|---|---|
| **ROT5** | digits 0-9 | `(d + 5) % 10` |
| **ROT13** | letters A-Z a-z | `(c + 13) % 26` |
| **ROT18** | digits + letters | ROT5(数字) + ROT13(字母) 组合 |
| **ROT47** | ASCII 33-126（94 个字符）| `(c - 33 + 47) % 94 + 33` |

**注意**：ROT47 是 **整段 ASCII 可打印字符旋转**，跟 ROT13（只字母）完全不同，是 CTF 隐藏 base64 解码后的字符串常用手段。

### 3.3 Base64 隐写原理（per [Welch 公开算法](https://www.cnblogs.com/cat54/p/11432478.html) + 我的理解）

**原理**：
- 标准 base64 编码：每 3 字节 → 4 字符（每个字符 6 bit → 共 24 bit = 3 字节）
- 但**末尾不足 3 字节**时：剩 1 字节 → 输出 2 字符 + `==`（实际只用 12 bit，**末 2 bit 冗余**）
- 剩 2 字节 → 输出 3 字符 + `=`（实际只用 18 bit，**末 4 bit 冗余**）
- **CTF 隐写**：把隐藏数据的 1 byte 拆成 4 个 2-bit，分别塞进 base64 末尾字符的**末 2 bit**（**每个原字符的末 2 bit 都藏 1 个 2-bit 数据**）
- 解码：取每字符末 2 bit → 4 个 2-bit → 拼成 1 byte → 这就是隐藏的数据

**算法**：
```python
def decode_base64_stego(s: str) -> bytes:
    """从 base64 字符串中提取隐藏的 2-bit 隐写数据。
    
    原理：base64 每字符 6 bit 表示数据，但实际只用前 4/2 bit（取决于对齐），
    末 2 bit 是冗余的。把这些末 2 bit 提取出来，4 个 2-bit 拼成 1 byte 隐藏数据。
    
    Args:
        s: 含隐写的 base64 字符串
    
    Returns:
        解出的隐藏 bytes (实际数据是 原始 base64 + 隐藏 bits)
    """
    # 1) 先正常 base64 decode 看长度
    decoded = base64.b64decode(s)
    # 2) 计算隐写容量
    # 每个 base64 字符末 2 bit → 4 个字符 → 1 byte 隐藏数据
    # 但要排除末尾 = 填充（不然末 2 bit 是填充位）
    stripped = s.rstrip('=')
    capacity = (len(stripped) * 2) // 8  # bits / 8 = bytes
    # 3) 提取末 2 bit
    bits = []
    for ch in stripped:
        v = _B64_INDEX[ch]
        bits.append((v >> 0) & 0b11)  # 末 2 bit
        # 注意：实际 base64 字符的 bit 顺序是 MSB first (位 5..0)
        # 这里取位 0-1（即末 2 bit），但要按正确顺序
    # 4) 4 个 2-bit → 1 byte
    hidden = bytearray()
    for i in range(0, len(bits) - 3, 4):
        byte = (bits[i] << 6) | (bits[i+1] << 4) | (bits[i+2] << 2) | bits[i+3]
        hidden.append(byte)
    return bytes(hidden)
```

> **重要**：base64 stego 需要 base64 **文本**作为输入（不是文件），所以 decoder 走 **text 模式**（弹 QInputDialog 取文本 / 或从 input 区读），**不走文件路径**。

### 6.2 GUI 弹 QInputDialog 的 UX

base64-custom 需要用户输入 64 字符表。3 种方案：
- **A**：弹单行文本输入框（用户粘贴 64 字符）→ 最简
- **B**：弹 ComboBox（"标准 / URL-safe / 自定义" 三选一）+ 自定义时弹文本框 → 中等
- **C**：直接读 input 区当前文本（如果 input 长度=64 就当作表）→ 最不打扰

→ **建议 B**（常用变体先列出来，节省用户输入）

### 6.3 base64 stego 编码要不要做？

Owner 只说要**解码**。但对称性考虑，编码（往 base64 里藏数据）也加进去，方便 reverse 题 / 出题。决策：
- **只做解码**：更符合 Owner 当前需求
- **解码 + 编码**：完整工具，但估时 +1h

→ **建议只做解码**，未来需要再加
