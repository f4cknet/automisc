# v0.5-tmp-text-mode — text 模式输出走 /tmp + GUI 弹 QFileDialog

> **创建时间**：2026-06-14 12:05
> **状态**：✅ done (commit (待 push))
> **触发**：Owner "输入 (7,7)\\n(7,8)\\n... 并点击坐标->二维码, 返回: 需要 --file <path> 决定 output 目录 (v0.5-output-samedir) ... 这是一个共性问题, 当用户输入的不是 file 而是文本时, 如果要输出文件都会存在相同的问题. 我的建议是, 如果输入不是文件路径, 而是文本文字, 那么如果输出文件, 统一输出到 /tmp/ 目录下. 而如果输入的是文件路径, 输出的文件则仍然和输入文件路径一样保持不变"

## 1. 问题

v0.5-output-samedir (per Owner 2026-06-14 09:27) 强制所有文件输出与 input 同目录——**但 text 模式没 input file 概念**：
- 用户粘 (7,7)(7,8)... 坐标串到 input 区
- 点菜单 [坐标→二维码]
- main_window 拿 `current_file` (e.g. meihuai.jpg) 当 file_path → 写到 `Challenge/meihuai__coords_qr.png`
- **误导的命名**——坐标串跟 meihuai 无关，但产物放它旁边

Owner 12:05 明确建议：**text 模式 (没 input file) → 走 /tmp；file 模式 (有 input file) → 仍 samedir**。

### 2.2 coords_to_qr runner 改造

**之前**：
```python
if not file_path:
    raise CoordsQRDecoderError("需要 --file <path>...")
out_path = output_path_for(file_path, suffix=".png", purpose="coords_qr")
```

**现在**：
```python
if file_path:
    out_path = output_path_for(file_path, ...)  # samedir (v0.5-output-samedir)
else:
    out_path = text_based_output_path(suffix=".png", purpose="coords_qr", out_dir=output_dir)
# + output_dir 参数 (CLI --out-dir / GUI QFileDialog)
```

_runner 接 `output_dir` kwarg 透传给 `decode_coords_to_qr`。

### 2.3 GUI 弹 QFileDialog (main_window.py)

text-based + 无 file_path 时，**弹 QFileDialog 让用户选 dir**：

```python
out_dir = QFileDialog.getExistingDirectory(
    self,
    f"选择 {decoder_name} 输出目录 (text 模式无 current_file)",
    str(self.output_dir_for_text_decoder),  # 上次选的 / 默认 cwd
)
```

新增 `self.output_dir_for_text_decoder` 记住用户上次选择（持久化在未来 v0.5+ 候选里）。

**用户取消** → 提示"已取消 (没选 output dir)"，不跑 decoder。

### 2.4 file 模式仍 samedir

v0.5-output-samedir 规则**没动**：
- 拖入文件 + 菜单触发 → current_file 存在 → samedir
- `automisc decode coords-qr --file <含坐标txt>` → 同上

只有**完全没 file_path**（纯 text 模式）才走 /tmp。
