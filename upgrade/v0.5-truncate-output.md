## 1. 问题

### 1.1 strings/grep 输出占满 GUI 窗口

**场景**：
- steg.png 233KB
- auto-run 触发 strings 工具
- `strings -a -n 4 steg.png` 输出 3398 行可打印字符串
- GUI `_on_runner_finished` 把 stdout 全部追加到 output 区 → **3398 行占满整个窗口**
- 但实际只命中 1 行（`L587: mAcTF`，base64 误报 ctf 关键字）
- 99.97% 是噪声

### 1.2 short-circuit 缺失

**问题**：未来 chain 嵌套场景（如 binwalk → lsb_extract → strings）下，lsb 命中后 strings 没必要再跑

**Owner 期望**：strings/grep 不匹配时**不打印 raw stdout**；lsb 命中后**链终止**

---

## 2. 修复

### 2.1 strings/grep 渲染版 stdout (v0.5-truncate-output)

**之前**：`ToolResult.stdout` 存 raw stdout（1000+ 行）
**现在**：`ToolResult.stdout` 存**渲染版**

**strings 渲染版格式**：
```
=== strings 摘要 (v0.5-truncate-output) ===
  total_lines: 3398
  suspicious:  1 (raw stdout 1000+ 行已省略, 只显示命中行)

--- 命中行 (max 20 显示) ---
  L587: mAcTF
```

无命中时：
```
=== strings 摘要 (v0.5-truncate-output) ===
  total_lines: 1000
  suspicious: 0
  └─ 无可疑特征, raw stdout 已省略 (想看原文: `strings -a -n 4 /path/to/file`)
```

**grep 渲染版格式**：类似，从 `offset` 字段提行号

### 2.2 AutoRunner short-circuit (v0.5-short-circuit-on-flag)

```python
SHORT_CIRCUIT_SEVERITY = 5

# 跑完每个工具后:
max_severity = max((sp.severity for sp in result.suspicious_points), default=0)
if max_severity >= SHORT_CIRCUIT_SEVERITY:
    self.short_circuited.emit(tool_name, f"命中 severity={max_severity}...")
    break
```

**新 signal** `short_circuited(tool_name, reason)` + `main_window._on_auto_short_circuited` 渲染：
```
[short-circuit] strings 命中 severity=5 (>= 5), 终止后续 tools
  └─ 后续 tools 跳过, auto-run 结束
```

**steg.png 真实场景**：
- auto-run 推荐 zsteg/steghide/exiftool/binwalk/strings
- strings 跑出 base64 误报 ctf 关键字（**severity=4 不触发 short-circuit**——base64/32/hex/binary=4 不到 5）
- **链正常跑完 5 个工具**——short-circuit 主要服务于"未来 chain 嵌套"场景

**steg.png 走菜单 lsb chain**：lsb_extract 命中 severity=5 → break（lsb chain 内部已有 Q3 终止逻辑）——这个**没动**

### 2.3 race fix (v0.5-short-circuit-fix 顺带)

3 个历史 runner e2e test 偶发 flake：等 `isRunning()=False` 后立刻 `toPlainText()`，但 slot 还没排到事件循环
- `test_main_window_lsb_chain_shows_full_text` (test_gui_bugfix.py)
- `test_menu_hex_ascii_uses_input_not_current_file` (test_input_output_view.py)
- 10x 重跑 9/10 → 0/10 全过
