## 1. 2 个 bug 现象

### Bug A: hex-ascii 工具卡死 + 乱码

**现象**：
- 拖入 meihuai.jpg
- 用户从 output 区复制 hex `28372c37290a...` 
- Clear + 粘贴 + 点击菜单栏 [hex-ascii]
- **它仍然去解 meihuai.jpg**（233KB 整个文件当 hex 解）
- 卡很久 + 输出 `���JFIF�Exif...` 乱码

**根因**：
- `core/decoders/base_convert.py` 的 `_runner(file_path)` 读整个文件当 hex 串
- `gui/main_window.py` 的 `_run_decoder` 没区分"text 模式"和"file 模式"decoder
- 菜单栏触发 `_run_decoder("hex-ascii")` → 把 `current_file` 传过去 → 整个 meihuai.jpg 被当 hex 解

### Bug B: 拖新文件不清空旧 output

**现象**：
- 拖 meihuai.jpg → output 出 hex
- 拖 steg.png → output 区**还有 meihuai 的残留**
- 用户看着乱

**根因**：
- `dropEvent` 没有 `self.output_view.clear()` 这一步
- 旧 runner 也没停（潜在并发跑两个文件 bug）

## 2. 修复

### 2.1 hex-ascii 改为 text-based decoder (v0.5-hex-ascii-fix)

**核心改动**：
- `core/decoders/base_convert.py` `_runner(file_path=None, text=None, **_)` 接受 text 优先
- `gui/decode_runner.py` `DecodeRunner(text=...)` 支持 text 模式 (file_path 用 `<text>` 占位)
- `gui/main_window.py` `_run_decoder` 加 `text_based_decoders = {"hex-ascii"}` 白名单
  - 走 text 模式: 从 input 区抽 candidate (复用 `extract_base_candidate()`)
  - 走 file 模式: 用 `current_file` (e.g. base64-image)
- **删掉 GUI 顶 bar 的 `[Hex → ASCII]` 按钮** (per Owner: 与菜单栏重复)
- `InputOutputView.extract_base_candidate()` 公共方法, 顶 bar 删除后给 main_window 复用
  - selection 优先 → 最后像 base 的行 → 最后非空行 (兜底)

**Size guard**：
- `MAX_INPUT_SIZE = 1MB` (hex-ascii 设计就是解短串, 大文件用 strings/foremost)
- 超 1MB 直接 raise `BaseConvertError`

**CLI 用法**：
```bash
# 1. 直接传 hex 串
$ automisc decode hex-ascii --text "48656c6c6f"
=== 🔢 Hex → ASCII ===
input: 48656c6c6f
detected_format: hex
output_text: Hello

# 2. 读含 hex 的文件 (e.g. 用户粘 hex 到 .txt)
$ automisc decode hex-ascii --file /tmp/hex.txt
```

### 2.2 拖入新文件清空旧 output (v0.5-clear-on-new-file)

**核心改动**：
- 抽出 `MainWindow._on_new_file_selected(file_path, source)` 统一入口
- `dropEvent` 和 `_open_file_dialog` 都走它
- 行为：1) 停旧 runner; 2) **清空 output**; 3) 设 current_file; 4) FileRouter; 5) auto-run

**GUI 端到端** (per Owner 09:50 场景):
```
1. 拖 meihuai.jpg -> auto-run -> [drop] file=meihuai.jpg
                     [auto-run] 跑 strings 出 hex
                     exit_code: 0
                     28372c37290a  <- 留在 output
2. 拖 steg.png    -> [新文件] steg.png  (v0.5-clear-on-new-file 已清空旧 output)
                     [drop] file=steg.png  <- 旧 hex 不见
                     [auto-run] 跑 strings / binwalk / ...
```
