## 1. 背景

### 1.1 Owner 反馈 (2026-07-01 09:02)

实战 `flag.pyc` 反编译跑通后, Owner 提 2 个改进:

1. **"工具栏分别针对 py2 和 py3 做 2 个 pyc 反编译"** —— 当前只有 1 个按钮 "🐍 Pyc 反编译 (默认 Python 2)", 按 magic 自动判断; Owner 想要**显式选 py2 还是 py3** (避免自动判断失误)
2. **"反编译成功后输出 py 文件到与输入文件 pyc 相同的目录"** —— 当前 `run_pyc_decompiler` **不写文件**, 只把 `source_code` 放进 `PycDecompileResult` 输出到 GUI output 区; Owner 想要**直接写盘**, 跟 `base64_image` / `coords_qr` / `magic_sniffer` 一致

### 1.2 当前问题

| 痛点 | 现状 | Owner 预期 |
|---|---|---|
| **强制版本** | `run_pyc_decompiler` 按 magic 自动判断, magic 错判时无手动 override 路径 | 工具栏 3 按钮: 自动 / py2 / py3 |
| **写盘** | 1 按钮反编译后**只显示在 output 区**, 要手工复制粘到 `flag.py` | 自动写 `flag__pyc_py2.py` 到 pyc 同目录 |
| **命名冲突** | 不写盘所以无冲突 | 3 按钮输出不同文件名 (py2/py3/auto) 不互相覆盖 |

### 1.3 关联历史

- **v0.5-pyc-magic-sniffer** (`17f0784`): 创建 `run_pyc_decompiler(file_path)`, 走 magic 自动判断, **不写盘**
- **v0.5-pyc-decompiler-gui** (`614cfdd`): 加到 GUI 工具栏 1 按钮 (显示名 "🐍 Pyc 反编译 (默认 Python 2)")
- **v0.5-pyc-deps-install** (`7577203`): `install.ps1` 装 uncompyle6 3.9.3 + decompyle3 3.9.3 + xdis 6.1.7
- **v0.5-output-samedir** (`af35fb0`): 抽 `core/utils/output_path.py:output_path_for(input, suffix, purpose)`, 命名 `<stem>__<purpose>.<ext>` 与 input 同目录 — 本 spec 复用

## 2. 目标

### 2.1 能力 A: GUI 工具栏 3 按钮 (per Owner 拍板)

| GUI entry | 显示名 | force_version | 写文件命名 |
|---|---|---|---|
| `decoder:pyc_decompiler` | 🐍 Pyc 反编译 (自动判版本) | `None` (走 magic) | `flag__pyc.py` |
| `decoder:pyc_decompiler:py2` | 🐍 Pyc 反编译 (强制 Python 2) | `2` (强制 uncompyle6) | `flag__pyc_py2.py` |
| `decoder:pyc_decompiler:py3` | 🐍 Pyc 反编译 (强制 Python 3) | `3` (强制 decompyle3) | `flag__pyc_py3.py` |

**实现机制**: menu_dock 用 entry 字符串后缀 `:py2` / `:py3` 携带参数, main_window `_run_decoder` 解析后缀 → DecodeRunner → `spec.run(force_version=N)` (per `inspect.signature` 自动 kwargs 透传机制, 跟 v0.5-base-rot-decoders `custom_table` 风格一致)

### 2.2 能力 B: 反编译成功后写 .py 到 pyc 同目录 (per Owner 拍板)

- **成功** (uncompyle6/decompyle3 反编译产出源码) → 写 `<stem>__pyc_pyN.py` 到 pyc 同目录
- **失败** (反编译抛异常 / method="dis" fallback) → **不写盘**, GUI output 区报错
- **写盘行为对所有 3 按钮统一** (auto / py2 / py3 都写)

**命名规范**: 跟 v0.5-output-samedir 风格一致 (`output_path_for(input, suffix, purpose)`):
- `flag.pyc` + `purpose="pyc"` → `flag__pyc.py`
- `flag.pyc` + `purpose="pyc_py2"` → `flag__pyc_py2.py`
- `flag.pyc` + `purpose="pyc_py3"` → `flag__pyc_py3.py`

### 2.3 不在本次范围

- ❌ **不动 `pyc_decompiler.py` 默认路由逻辑** (magic 自动判断仍可用, `force_version=None` 走原路径)
- ❌ **不引入新依赖** (xdis 6.1.7 + uncompyle6 3.9.3 + decompyle3 3.9.3 已在 v0.5-pyc-deps-install 装好)
- ❌ **不做 GUI 参数 dialog** (per v0.5-pyc-magic-sniffer spec §4.2; 3 按钮走硬编码)
- ❌ **不修 uncompyle6 反编译顶层 consts bug** (per v0.5-train-019 §3.4, 单题实战命中, 不升架构)
- ❌ **不做批量 pyc 目录遍历** (单文件, Owner 实战场景)

## 3. 设计

### 3.1 改 `core/decoders/pyc_decompiler.py`

#### 3.1.1 `PycDecompileResult` 加 2 字段

```python
@dataclass
class PycDecompileResult:
    input_path: str
    raw_size: int = 0
    source_code: str = ""
    method: str = ""  # "uncompyle6" / "decompyle3" / "dis"
    magic_int: int = 0
    version: tuple = field(default_factory=tuple)
    error: Optional[str] = None
    # v0.5-pyc-decompiler-buttons 新增
    force_version: Optional[int] = None  # 实际 force 的版本 (None = auto)
    output_path: Optional[str] = None  # 写盘后的 .py 路径 (None = 没写)

    @property
    def success(self) -> bool:
        # 写盘后才算 success (per Owner "反编译成功后输出 py 文件")
        return self.output_path is not None and self.error is None
```

#### 3.1.2 `run_pyc_decompiler` 签名扩展

```python
def run_pyc_decompiler(
    file_path: str,
    force_version: Optional[int] = None,  # None=auto / 2=py2 / 3=py3
    write_output: bool = True,  # 是否写盘 (默认 True, CLI 可关)
    output_dir: Optional[str] = None,  # None=pyc 同目录 / str=指定 dir
) -> PycDecompileResult:
```

#### 3.1.3 路由逻辑

```python
# 当前 (v0.5-pyc-magic-sniffer):
if is_py2:
    source, method = _decompile_with_uncompyle6(file_path)
elif is_py3:
    source, method = _decompile_with_decompyle3(file_path)
else:
    err = f"unsupported Python version: {version}"

# 本 spec (force_version 覆盖):
if force_version == 2:
    # 强制 py2, 跳过 magic 判断
    source, method = _decompile_with_uncompyle6(file_path)
elif force_version == 3:
    # 强制 py3, 跳过 magic 判断
    source, method = _decompile_with_decompyle3(file_path)
elif is_py2:
    source, method = _decompile_with_uncompyle6(file_path)
elif is_py3:
    source, method = _decompile_with_decompyle3(file_path)
else:
    err = f"unsupported Python version: {version}"
```

**注意**: `force_version=2` 强制 uncompyle6 (即便 magic 是 Py3.10) — Owner 偶尔有需要"用 Py2 工具试 Py3 字节码"逆向场景

#### 3.1.4 写盘逻辑

```python
# 写盘: 成功反编译 (source 非空) + write_output=True
if source and write_output:
    # 选 purpose 后缀
    if force_version == 2:
        purpose = "pyc_py2"
    elif force_version == 3:
        purpose = "pyc_py3"
    else:
        purpose = "pyc"
    # output_path_for 复用 v0.5-output-samedir
    out_path = output_path_for(p, suffix=".py", purpose=purpose)
    if output_dir:
        # GUI 弹 QFileDialog 选 dir (v0.5-tmp-text-mode-2 风格)
        out_path = Path(output_dir) / f"{p.stem}__{purpose}.py"
        out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(source, encoding="utf-8")
    output_path = str(out_path)
else:
    output_path = None
```

## 6. 风险 + 缓解

| 风险 | 缓解 |
|---|---|
| `force_version=2` 强制 uncompyle6, magic 实际是 Py3.x → 反编译必然失败 | GUI output 区报 `uncompyle6 failed: ...` 不写盘, Owner 自查 |
| `output_path_for` 写盘到 pyc 同目录, 但 input 目录无写权限 (Win 11 受限目录) | 写盘失败 try/except, 写 `result.error = f"write file failed: {e}"`, GUI output 区报红 |
| 3 按钮命名冲突, Owner 容易点错 | 显示名明确 "自动判版本" vs "强制 Python 2/3", 强语义区分 |
| `:py2` / `:py3` 后缀跟未来其他 decoder param 冲突 | 命名规范统一 `:param=value` 风格 (本 spec 引入), menu_dock entry 解析统一在 main_window |
| uncompyle6 反编译顶层 consts bug (per v0.5-train-019) | 不修, 训练日志记录, owner 实战手工修 (per §5.2 <3 同类) |
