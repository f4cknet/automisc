## 1. 背景

### 1.1 当前行为（v0.5-DAG-chain-realtest 后）

`Challenge/steg.png` 跑通流程：
1. `automisc chain --chain foremost` → 1 步，提取原 PNG（无嵌入）→ 用户看 output "foremost 没找到"
2. 用户看到 router 推荐 `zsteg` 跑得动 → 单独跑 `zsteg Challenge/steg.png`
3. zsteg 输出 `b1,rgb,lsb,xy .. text: "Hey I think we can write safely in this file without anyone seeing it. Anyway, the secret key is: st3g0_saurus_wr3cks"`
4. **当前 adapter**：把这段 text 标 severity=4 SuspiciousPoint
5. **期望行为**：adapter 抽出 raw text → **识别** "secret" / "key" 关键词 → **打印 "找到疑似 flag: st3g0_saurus_wr3cks"** → **终止链**

### 1.2 Owner 提的关键 insight

> "这里的逻辑是 LSB 的识别逻辑，应该是判断纯 text 还是图片（gif,png,jpg）通过文件头比如 89504E47、压缩包（rar,zip）通过文件头比如 504B0304，如果是纯 text 直接打印并停止，如果是文件则进入下一步流程，比如图片内容识别，比如 zip 伪加密、4-6 位数暴力破解，或者是 rar 的 4-6 位的暴力破解"

→ **LSB 抽出后要当 router 用**——判断"抽出来的东西"是什么类型，路由到对应处理。

## 2. 设计决策（Owner Q&A）

### Q1: 什么样的 text 才标严重 / 终止？

**Owner 答**（最终版）：
- **敏感关键词**（大小写不敏感）：`key` / `flag` / `secret` / `ctf` / **`password`**（后补）→ **severity=5**
- **编码特殊形态**：
  - base64 字符串（核心字符 ≥ 12 + 可解码）→ severity=4
  - base32 字符串（核心字符 ≥ 12）→ severity=4
  - 纯 binary 串（0/1 chars，长度 ≥ 32）→ severity=4
  - 纯 hex 串（0-9a-f，长度 ≥ 32）→ severity=4
  - 一般英文 / 中文文本 → severity=3（让用户自己看，不强标）

**打印策略（Owner 简化）**：
- text 找到 → **都打印**（命中/不命中都打印）
- 区别只在于 **"高亮"**：命中 → `[高亮] 命中敏感关键词`；不命中 → 普通打印
- **不抽 flag_candidate 字符串**（命中后**整段 text 就是 candidate**；不命中 → None）

### Q2: 抽到 file 怎么递归？

**Owner 答**：按 Mavis 建议——`binwalk_extract` 链尾部加 `LSBExtractAction`：
```
binwalk 检测 → foremost 提取 → zsteg 检测 → 抽到 file 再喂 router
```
新 chain `build_lsb_extract_chain` 复用 router 自动分流。

### Q3: 终止条件？（Owner 后续追加 + 经验修正）

**Owner 答**（最终版）：
- **Owner 经验**：LSB 通道有纯文本 → 其他通道通常无新线索
- **最终行为**：
  1. 扫所有 text 通道（zsteg 默认会列 N 个）
  2. 逐个抽 raw + 算 severity
  3. **命中 severity=5**（敏感词）→ 立即停（**`lsb_texts_scanned`** 含扫描过的所有通道，GUI 可显示）
  4. **都没命中**（只有 severity 3/4）→ 按 zsteg 顺序打印第一个 + 停
- **简化方向**：你提的 "if ["key","flag","secret","ctf","password"] in text" — 直接子串匹配，**不抽 key/flag 字符串**，**命中后整段 text 即 candidate**

### Q4: rar 爆破？

**Owner 答**：**Mavis 决定**——john 通用 password attack（per v0.5+ 已规划 PR5 计划）能打 rar，**已有**。本迭代不引入新工具，复用 john 即可。

## 3. 架构

### 3.1 新增模块

```
src/automisc/core/actions/lsb_extract.py       ← 新 (v0.5-LSB-router 核心)
  - LSBExtractAction: 调 zsteg -e <channel> <file> 抽出 raw bytes
  - 内置 _classify_extracted(bytes) → 分类
    - "text"      (utf-8 decodable + 短)
    - "base64"    (高 entropy 字符串)
    - "binary"    (raw bytes 不 decodable)
    - "file"      (已知 magic header)
  - 命中敏感关键词 + file → action 把 raw bytes 写到 tmp + 调 router.route(tmp) 二次分诊

src/automisc/core/encoding_detector.py        ← 新 (Q1 编码检测)
  - is_base64(s) → bool (高 entropy + 长度)
  - is_binary_string(s) → bool (0/1 字符)
  - has_sensitive_keyword(s) → bool (secret/key/flag/ctf)
  - score_text_severity(text) → 3|4|5

src/automisc/core/magic.py                     ← 新 (Q2 复用)
  - detect_magic_header(data) → str (e.g. "PNG image" / "ZIP archive" / "RAR archive")
  - 复用 router.py 的 detect_magic
```

### 3.2 新增 chain

```
src/automisc/core/chains.py
  + build_lsb_extract_chain(file_path): DAG
    1. BinwalkExtractAction: binwalk 检测 + foremost 提取
    2. (并行) LSBExtractAction: zsteg -e 各通道
       - 抽出 text: 分类 → 终止 + 打印
       - 抽出 file: 写 tmp + router 二次分诊 → 触发 zip_chain / rar_chain / png_chain
    3. DAG 终止条件: 找到 text 终止 / 找到 file 递归（限 max_depth=3 防死循环）
```

### 3.3 行为表

| zsteg 输出 | 抽出 raw 后 | 动作 | severity |
|---|---|---|---|
| `b1,rgb,lsb,xy .. text: "flag{abc}"` | text (含 flag 关键词) | **打印 + 终止** | 5 |
| `b1,rgb,lsb,xy .. text: "secret key: xyz"` | text (含 secret/key 关键词) | **打印 + 终止** | 5 |
| `b1,rgb,lsb,xy .. text: "aGVsbG8gd29ybGQ="` | base64 (可解码 + 长度 ≥ 16) | 打印 + 提示"疑似 base64" | 4 |
| `b1,rgb,lsb,xy .. text: "010101001110..."` | binary string | 打印 + 提示"疑似 binary" | 4 |
| `b1,rgb,lsb,xy .. text: "Hey I think..."` | 普通英文 | 打印 + 让用户判断 | 3 |
| `b1,rgb,msb,xy .. file: Zip archive data` | raw = ZIP magic | 写 tmp + router → **zip_chain 递归** | 5 |
| `b1,rgb,msb,xy .. file: RAR archive data` | raw = RAR magic | 写 tmp + router → **rar_chain (john)** | 5 |
| `b1,rgb,msb,xy .. file: PNG image data` | raw = PNG magic | 写 tmp + router → **png 链递归**（foremost + zsteg）| 5 |

## 4. 决策树（Owner 最终版）

```
LSBExtractAction.run(file_path)
  1. 调 `zsteg <file>` 找出所有 text / file 行
  2. text 行处理 (Q3 简化 + Owner 经验):
     a) 扫所有 text 通道 (zsteg 顺序, b1,rgb,lsb,xy 在前)
     b) 抽 raw text (zsteg -e <channel> <file>) + 算 severity
     c) 命中 severity=5 (key/flag/secret/ctf/password) → 立即停
     d) 都未命中 → 按 zsteg 顺序返回第一个 text + 停
     e) flag_candidate = 整段 text (命中) / None (未命中)
     f) lsb_texts_scanned 含所有扫过的通道 (GUI 可展示)
  3. file 行 (Q2 二次 router):
     a) 抽 raw file (zsteg -e <channel> <file>)
     b) 写 tmp + router.route(tmp) 二次分诊
     c) 触发 zip_chain / rar (john) / png (递归)
  4. 递归深度保护: max_depth=3
```

## 5. 实现细节

### 5.1 编码检测阈值

```python
def score_text_severity(text: str) -> int:
    if has_sensitive_keyword(text):  # secret|key|flag|ctf
        return 5
    if is_base64(text, min_len=16):
        return 4
    if is_binary_string(text, min_len=32):  # 0/1 chars
        return 4
    if is_hex_string(text, min_len=32):  # 0-9a-f
        return 4
    return 3
```

## 7. 风险

| 风险 | 缓解 |
|---|---|
| 递归死循环（LSB 抽 PNG → 抽 PNG → ...）| `max_depth=3` 强限制 + 每步 ctx 标记 depth |
| base64 误判（普通文本里出现 "aGVsbG8="）| 长度 ≥ 16 + 字符集匹配 + 可解码 三重验证 |
| zsteg 抽出后 magic 不全（zip 部分损坏）| router 二次分诊，magic 复用 |
| text 终止过早（hint 没看完就停）| **Q3 Owner 决策**：text 立即停（用户决定下一步）|
