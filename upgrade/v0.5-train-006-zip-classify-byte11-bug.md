## 3. automisc 当前 bug

### 3.1 算法层 bug — `_classify_zip_entries` byte[11] 启发式错误

源码 `src/automisc/core/actions/zip_chain.py:124-131`：

```python
# 看 data 起始 12 字节末位 (PKCS#5 末字节) 是否在 0-11
last_byte = data[data_start + 11]
if last_byte in range(12):
    # 有 PKCS#5 header → 真加密 (修了也白修, data 仍加密)
    real[fname] = (lfh_offset, cdh_offset)
else:
    # 无 PKCS#5 header → 伪加密
    pseudo[fname] = (lfh_offset, cdh_offset)
```

**为什么是错的**：

- 真 PKCS#5/ZipCrypto 加密后, byte[11] 是密文, **可以是 0-255 任意值**
- 该字节解密前的明文是 CRC-32 高字节, 但解密后经过 PKCS#5 update + XOR, 密文分布**接近均匀随机**
- 启发式命中率: `12/256 ≈ 4.7%` — **绝大多数真加密被错判成伪加密**

## 4. 正确判定逻辑 (per ctf-wiki 加密原理)

**核心思路**: 不要看 byte[11], 要看 **deflate 是否能解**。

| 场景 | 直接 inflate | 跳过 12 字节 inflate | CRC 校验 | 判定 |
|---|---|---|---|---|
| 伪加密 (flag 改了, data 明文 deflate) | ✅ 成功 | (跳过必失败) | 匹配 header CRC | **pseudo** |
| 真加密 (flag 设了, data 是加密 deflate) | ❌ 失败 | ❌ 失败 (无密码解不开) | N/A | **real** |
| 真加密但运气好密码已知 | ❌ 失败 | ✅ 成功 (解出原 deflate) | 匹配 | **real** |
| 损坏 / 截断 | ❌ 失败 | ❌ 失败 | N/A | **real** (保守) |

**保守归类原则**: 不能 deflate + CRC 校验时, 默认归 **real** (不归 pseudo):
- pseudo → user 跑 ZipCenOp r (破坏性, 改字节后**不能**验证是否真解出)
- real → user 跑 bruteforce (非破坏性, 跑 1 分钟没结果就走密码学/手工)
- **误判 real 比 误判 pseudo 安全**: real 让用户保留文件, pseudo 误导用户做破坏性修改

## 5. 修复方案 (架构升级)

修改 `_classify_zip_entries` 第三阶段判定逻辑 (`src/automisc/core/actions/zip_chain.py`)：

```python
# 新判定 (替代 byte[11] in range(12)):
import zlib

# ... 在 for fname, (lfh_offset, lfh_flag, comp_size, data_start) in lfh_map.items(): 内

if comp_size < 12:
    pseudo[fname] = (lfh_offset, cdh_offset)
    continue

# 读 header CRC 用于验证
crc = struct.unpack("<I", data[lfh_offset + 14:lfh_offset + 18])[0]
file_data = data[data_start:data_start + comp_size]

# 1) 直接 inflate (无 header)
try:
    decompressed = zlib.decompress(file_data, -15)  # raw deflate
    if zlib.crc32(decompressed) & 0xFFFFFFFF == crc:
        pseudo[fname] = (lfh_offset, cdh_offset)
        continue
except zlib.error:
    pass

# 2) 跳过 12-byte PKCS#5 header 再 inflate
if comp_size > 12:
    try:
        decompressed = zlib.decompress(file_data[12:], -15)
        if zlib.crc32(decompressed) & 0xFFFFFFFF == crc:
            real[fname] = (lfh_offset, cdh_offset)
            continue
    except zlib.error:
        pass

# 3) 都失败 → 保守归 real (per §4 原则)
real[fname] = (lfh_offset, cdh_offset)
```

**为什么**用 CRC 校验而不是只靠 inflate 成功:
- 极小概率: 随机 29 字节数据被 zlib 接受为合法 deflate 流 + CRC 巧合匹配 — 不可能 (CRC-32 碰撞)
- CRC 校验是 ZIP 标准自带的字段, 不用密码也能拿到, 是真加密/伪加密的客观判据
