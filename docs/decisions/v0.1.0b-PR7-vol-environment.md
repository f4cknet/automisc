## 最终方案

### vol3（**adapter 默认走它**）
- `pip install volatility3`（已装）
- 入口：`python -c "from volatility3.cli import main; main()"`
- 支持的 profile（自动检测）：win10 / win11 / Linux / macOS
- 优点：Python 3 原生、官方持续维护
- 缺点：少数老 plugin（malfind / pslist 的某些 flag）行为与 vol2 微差异

### vol2（**Docker 包装 / 用户手动调**）
- **当前不在 v0.1 adapter 链路里**——Python 2 源码无法 pip install
- 源码在 `extend_tools/volatility2/`（per §6.1 处置规则，PR7-envfix **不删**，留 v0.5+ Docker 化）
- v0.5+ 计划：写一个 `vol2_docker.sh` wrapper（`docker run --rm -v $PWD:/data phocean/volatility`）作为 backup
- GUI 留 2 个 menu entry：vol3 adapter + vol2 docker wrapper
